# 弹性潜在接口：让扩散模型灵活计算的新范式

**论文：** One Model, Many Budgets: Elastic Latent Interfaces for Diffusion Transformers  
**arXiv：** 2603.12245  
**机构：** Rice University, Snap Inc.  
**日期：** 2026年3月  

---

## 一、背景与动机

扩散变压器（Diffusion Transformers, DiTs）近年来在图像和视频生成领域取得了显著突破，凭借其简洁的设计和出色的可扩展性，已成为生成模型的主流架构之一。然而，随着模型规模的不断增大，计算成本也随之水涨船高，一个核心问题逐渐浮现：**DiTs 的计算效率是否被充分利用了？**

论文作者通过精心设计的实验揭示了两个关键问题：

**1. 计算与分辨率的刚性绑定**  
传统 DiT 的每步计算成本固定地取决于输入分辨率，无法根据延迟或计算约束灵活调整。这意味着即使你愿意等待更长时间获得更高质量，或者希望快速获得一个"够用"的结果，标准 DiT 都无法提供这种灵活性。

**2. 空间上的均匀计算分配**  
更深层的问题在于，DiT 将计算资源均匀分配到所有空间 token 上。作者设计了一个巧妙的实验：将真实图像用零值填充（padding），人为增加 token 数量。结果发现，即使提供了 4 倍的计算量，DiT 的生成质量完全没有提升——注意力图显示，零填充区域的 token 主要与其他零值区域交互，浪费了计算资源。

这揭示了一个根本性缺陷：**图像的不同区域难度不同**（有的区域简单，有的复杂），但 DiT 无法将计算从简单区域重新分配到困难区域。这就像一个学生无论题目难易都花同样的时间，效率自然低下。

## 二、核心方法：弹性潜在接口（ELIT）

针对上述问题，作者提出了**弹性潜在接口变压器（Elastic Latent Interface Transformer, ELIT）**，一个极简但强大的解决方案。

### 2.1 架构设计

ELIT 的核心思想是引入一个**可变长度的潜在 token 序列**，称为"潜在接口"（latent interface），并配备两个轻量级交叉注意力层：

- **Read 层**：从空间 token 中"拉取"信息到潜在接口，优先处理困难区域
- **Write 层**：将更新后的潜在信息"广播"回空间 token

整个架构分为三段：
1. **空间头部（Spatial Head）**：少量 transformer 块处理原始 patchified 输入
2. **潜在核心（Latent Core）**：大部分 transformer 块在潜在空间操作
3. **空间尾部（Spatial Tail）**：少量块完成最终输出

这种设计的关键优势是：**潜在接口的长度由用户控制，直接决定每步的计算量**，而与输入分辨率解耦。

### 2.2 分组交叉注意力

为了降低计算成本，ELIT 采用分组策略：
- 将空间 token 划分为 G 个不重叠的组（如 16 个 2D 网格）
- 潜在 token 也相应分组，每组 J = K/G 个
- 交叉注意力仅在组内进行，将复杂度从 O(NK) 降至 O(NK/G)

更重要的是，**组数 G 可以随分辨率变化，但每组潜在 token 数 J 保持不变**，这消除了对固定输入分辨率的依赖。

### 2.3 多预算训练与重要性排序

ELIT 的真正威力在于训练策略：

**随机尾部丢弃（Tail Token Dropping）**  
训练时，每个迭代随机采样 J̃ ∈ [J_min, J_max]，只保留每组前 J̃ 个潜在 token，丢弃后续的尾部 token。这创造了重要性层次：
- **头部 token** 训练次数多，被迫存储最重要信息（全局结构）
- **尾部 token** 训练次数少，用于精细化细节

推理时，J̃ 成为一个**用户可控的计算旋钮**：选择更少的 token 降低计算量，选择更多 token 提升质量，实现平滑的质量-计算权衡。

### 2.4 廉价分类器自由引导（CCFG）

ELIT 还解锁了一个意外的能力：**原生支持自引导（Autoguidance）**。

传统的分类器自由引导（CFG）需要调用两次模型（条件和无条件），计算量翻倍。自引导用一个弱模型替代无条件模型，但需要单独训练或手工构造弱模型。

而在 ELIT 的多预算框架下，**弱模型唾手可得**：只需用更少的潜在 token（J̃_w < J̃）运行同一个模型！作者进一步结合 CFG 和自引导的优势，提出**廉价 CFG（CCFG）**：

```
v̂_CCFG = (λ+1)G(X_t|c; J̃) - λG(X_t|∅; J̃_w)
```

这既提升了质量，又将计算成本降低约 33%，无需重新训练或手工调参。

## 三、实验设计

### 3.1 基线与设置

作者在多个主流扩散架构上验证 ELIT：
- **DiT**：标准扩散变压器
- **U-ViT**：带 U-Net 风格跳跃连接的 DiT
- **HDiT**：通过 PixelShuffle 减少中间 token 的 DiT
- **MM-DiT**：多模态扩散变压器（Qwen-Image）

**数据集：**
- 图像：ImageNet-1K（256px 和 512px）
- 视频：Kinetics-700（29 帧，256px）

**评估指标：**
- FID（Fréchet Inception Distance）
- FDD（DINOv2 特征上的 Fréchet Distance）
- IS（Inception Score）

所有基线使用相同的 transformer 块、相同的 Rectified Flow 训练目标、相近的训练计算量，确保公平比较。

### 3.2 关键实验

**实验 1：计算重分配验证**  
使用合成数据集（真实图像 + 零填充），验证 ELIT 能否将计算从零值区域重分配到真实区域。

**实验 2：质量-计算权衡**  
通过改变潜在 token 数量，评估 ELIT 在不同计算预算下的性能，并与减少采样步骤的策略对比。

**实验 3：大规模模型验证**  
在 20B 参数的 Qwen-Image 上应用 ELIT，验证其在超大规模模型上的可行性。

## 四、结果与分析

### 4.1 计算重分配的成功

实验结果清晰地验证了 ELIT 的核心假设：
- **DiT-B/2-Synth**（零填充版本）的性能与原始 DiT-B/2 完全相同，即使 token 数量增加了 4 倍
- **ELIT-DiT-B/2-Synth** 成功利用额外计算，达到与真实数据基线相当的质量

注意力图分析揭示：ELIT 的 Read 层能够智能地聚焦于信息丰富的空间区域（高 loss 区域），有效过滤零值区域。

### 4.2 惊人的性能提升

**ImageNet-1K 256px（单预算训练）：**
- ELIT-DiT vs DiT：FID 从 13.0 降至 8.2（-37%）
- ELIT-U-ViT vs U-ViT：FID 从 8.3 降至 7.5（-10%）
- ELIT-HDiT vs HDiT：FID 从 12.8 降至 9.4（-27%）

**ImageNet-1K 512px（多预算训练，ELIT-MB）：**
- DiT：FID 降低 46%，FDD 降低 52%
- U-ViT：FID 降低 34%，FDD 降低 33%
- HDiT：FID 降低 26%，FDD 降低 34%

**训练收敛速度：**
- 256px：3.3 倍加速
- 512px：4.0 倍加速

### 4.3 优于减少采样步骤

实验显示，通过减少潜在 token 数量来降低计算，比简单地减少采样步骤提供了更好的质量-计算权衡。更重要的是，这两种策略可以**组合使用**：对于每个计算预算目标，最优的步骤数和 token 数组合各不相同。

### 4.4 大规模模型的验证

在 20B 参数的 Qwen-Image 上：
- ELIT 将采样计算量降低至 35%-69%（取决于 token 数量）
- 在 DPG-Bench 上，平均分从 90.45（100% token）到 88.02（12.5% token）平滑过渡
- 结构细节得以保留，主要变化在于高频纹理细节

### 4.5 消融研究的关键发现

**分组大小：** 16 组（4×4 网格）效果最佳。过小的组（1×1）强制刚性的一对一映射，过大的组（16×16）缺乏空间正则化。

**块分配：** 约 67-71% 的块应在潜在核心，其余分配给头部和尾部。DiT-XL 采用 4-20-4 配置。

**多预算策略：** 联合训练的多预算模型优于独立训练的单预算模型，证明多预算训练起到了正则化作用。

## 五、评价与思考

### 5.1 核心贡献

ELIT 的最大价值在于其**简洁性与有效性的完美平衡**：
- 仅添加两个交叉注意力层
- 保持原有的 Rectified Flow 目标不变
- 可无缝集成到多种架构（DiT、U-ViT、HDiT、MM-DiT）

这种"极简干预"的设计哲学值得借鉴——与其设计复杂的新架构，不如在现有架构上找到关键的改进点。

### 5.2 理论洞察

论文揭示的"空间均匀计算"问题具有普遍意义。不仅 DiT，许多视觉 transformer 可能都存在类似问题。ELIT 的潜在接口提供了一种通用机制，让模型**学会**如何分配计算，而非被动地均匀分配。

重要性排序的训练策略（尾部丢弃）也很有启发性：它创造了一种自然的课程学习，让模型从粗到细地学习表示。

### 5.3 局限性与未来方向

论文坦诚地指出了几个局限：
1. **大规模从头训练未验证**：当前结果基于中等规模或微调实验，超大规模预训练的收益有待验证
2. **CCFG 的饱和问题**：CCFG 在高引导尺度下更容易产生过饱和图像，建议使用较低的引导权重
3. **噪声级别感知的预算调度**：初步实验表明高噪声步骤可能需要更少 token，但这方面研究尚浅

未来值得探索的方向：
- 训练和推理时的动态预算调度器
- 结合其他加速技术（如 TeaCache、token merging）
- 扩展到更多模态（文本、音频）

### 5.4 与相关工作的对比

**vs. 自适应生成器（如 Slimmable Networks）：**  
ELIT 不仅支持多预算推理，还通过重要性排序提升了固定预算下的性能，这是单纯的多分支网络无法实现的。

**vs. Token Dropping（如 MaskDiT、TREAD）：**  
这些方法在训练时丢弃 token 以加速，但推理时仍使用全量 token（因为信息损失不可恢复）。ELIT 通过潜在接口避免了这个问题，推理时可以自由调整 token 数量。

**vs. RINs/FITs：**  
这些方法也使用潜在 token，但架构与 DiT 差异较大，需要特殊优化器。ELIT 保持了与 DiT 的高度兼容性，更易采用。

## 六、适用场景

### 6.1 推荐场景

1. **高分辨率生成任务**  
   随着分辨率提高，像素冗余增加，ELIT 的动态计算重分配优势更加明显。512px 上的提升显著大于 256px。

2. **延迟敏感应用**  
   实时或交互式场景（如游戏资产生成、实时视频处理）需要在严格延迟约束下提供最佳质量。ELIT 允许根据当前负载动态调整计算预算。

3. **边缘设备部署**  
   资源受限设备（手机、IoT）可以通过减少潜在 token 数量，在有限算力下运行原本需要更大模型的任务。

4. **多质量服务**  
   同一个模型可以服务不同付费级别的用户（低预算用户获得快速但质量稍低的生成，高预算用户获得高质量生成），无需维护多个模型。

5. **研究与实验**  
   快速迭代实验时，可以用较少 token 进行初步验证，确定方向后再用更多 token精细化。

### 6.2 不适用场景

1. **极致质量追求**  
   如果应用场景对质量要求极高且计算资源充足，传统全量计算可能仍是最优选择。

2. **超低延迟场景**  
   虽然 ELIT 可以减少计算，但交叉注意力层引入了额外开销。在毫秒级延迟要求下，可能需要更激进的优化。

3. **非 transformer 架构**  
   ELIT 专为 transformer 设计，对于基于 CNN 的扩散模型（如 U-Net），需要重新设计潜在接口机制。

---

## 七、总结

**弹性潜在接口（ELIT）** 是扩散模型效率优化领域的一个重要里程碑。它通过一个极简的设计（两个交叉注意力层 + 尾部丢弃训练），解决了 DiT 的两个根本问题：计算与分辨率的刚性绑定、空间上的均匀计算分配。

实验结果令人印象深刻：
- **35-40% 的 FID/FDD 提升**
- **3-4 倍的训练加速**
- **平滑的质量-计算权衡**
- **原生支持自引导，额外 33% 推理加速**

更重要的是，ELIT 展示了一种新的设计哲学：**让模型学会如何分配计算**，而非依赖手工规则或复杂架构。这种思路很可能启发生成模型之外的更多应用。

对于实践者，ELIT 提供了一个低风险、高回报的改进方案——只需少量代码修改，即可在现有 DiT 架构上获得显著提升。对于研究者，ELIT 开启了"弹性计算"这一新方向，许多有趣的扩展值得探索。

**一句话总结：** ELIT 证明了，有时候最简单的改变，能带来最深远的影响。
