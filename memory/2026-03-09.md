## Kimi CLI 配置

### 安装
从 VSCode 扩展里提取 kimi-cli：
```bash
# 解压到永久位置
mkdir -p ~/.local/lib
cd /tmp && tar -xzf ~/.vscode-server/extensions/moonshot-ai.kimi-code-0.4.4-linux-x64/bin/kimi/archive.tar.gz
mv /tmp/kimi ~/.local/lib/kimi
ln -sf ~/.local/lib/kimi/kimi ~/.local/bin/kimi

# 添加到 PATH
echo 'PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

### 网络问题排查
**问题：** `kimi login` 连接超时

**原因分析（感谢 Copilot）：**
1. Windows 的 `HTTPS_PROXY=127.0.0.1:7897` 继承到 WSL2，但 WSL2 里这个端口不存在
2. WSL2 eth0 MTU=9000（巨帧）与 Clash TUN MTU=1500 不匹配，TLS 握手大包被丢弃

**修复：**
- `~/.bashrc` 添加 `unset HTTP_PROXY HTTPS_PROXY http_proxy https_proxy`
- systemd 服务自动设置 eth0 MTU=1500

**关键点：** Windows Clash TUN 模式已经透明代理，不需要设置代理环境变量
