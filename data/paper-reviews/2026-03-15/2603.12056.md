# XSkill: 多模态Agent的经验与技能持续学习框架

## 背景与动机

随着多模态大语言模型(MLLMs)的快速发展,AI Agent已经从被动感知系统进化为主动问题解决者,能够通过多样化的工具处理复杂的推理任务。然而,当前的多模态Agent在开放环境中仍面临两个根本性瓶颈:

**问题一:工具使用效率低下**
Agent往往在简单问题上花费过多步骤,而在面对复杂查询时又无法进行足够深入的多轮探索。这种低效性不仅浪费计算资源,还可能导致任务失败。

**问题二:工具编排缺乏灵活性**
大多数现有系统局限于单路径执行,在跨任务组合工具方面表现出明显的局限性。固定的工作流程无法适应复杂多变的多模态场景。

更关键的是,现有的持续学习方法主要依赖文本轨迹日志进行知识提取和检索,这在关键决策信号往往根植于视觉观察的多模态环境中是远远不够的。没有将经验和技能锚定在任务的视觉状态上,Agent就无法可靠地检索相关的先验知识或将其适应当前的执行上下文。

人类自然地通过从经验和技能中持续学习来提升问题解决能力。经验捕捉从先前尝试中提炼出的简洁且上下文敏感的指导,帮助优化局部决策如工具选择、探索和错误恢复。技能则编码更加结构化和可复用的程序,支持跨相关任务的高层规划和工具编排。这两种知识形式共同支持高效的执行和灵活的问题解决。

基于此洞察,论文提出XSkill框架,这是首个统一视觉锚定的任务级技能和动作级经验的双流设计,使多模态Agent能够从视觉-工具交互中进行无参数更新的知识积累。

## 核心方法

XSkill采用双流架构,将知识结构化为双层表示:

### 1. 知识表示

**技能(Skills)**
- 提供任务级指导,包含结构化工作流程
- 形式化表示:k = (M, W, P)
  - M: 元数据(技能名称、描述、版本号)
  - W: 工作流程序列
  - P: 可复用工具模板
- 存储为结构化Markdown文档

**经验(Experiences)**
- 提供动作级洞察,捕捉高层指令中常被忽略的战术知识
- 形式化表示:e = (c, a, v_e)
  - c: 触发条件
  - a: 推荐动作
  - v_e: 语义嵌入(用于检索)
- 存储为JSON格式的经验库
- 文本长度限制:|c| + |a| ≤ L_max^e

### 2. 双阶段架构

**Phase I: 知识积累阶段**

针对训练数据集D_train,Agent执行N次独立rollout生成轨迹集R_i。通过三个关键步骤提取和整合知识:

1. **Rollout摘要**
   - MLLM_kb执行视觉引导的rollout摘要化
   - 输入:轨迹集R_i、任务图像I_i、查询q_i、真实答案y_i*、适配的技能K_adapted
   - 输出:轨迹摘要S_Ri和技能片段ΔK_i
   - 将知识与视觉观察锚定,而非仅依赖文本轨迹日志

2. **跨Rollout批判**
   - 对成功和失败轨迹进行对比分析
   - 识别结果背后的因果因素
   - 输出结构化经验更新ΔE_i = {op_1^e, op_2^e, ..., op_Mi^e}
   - 操作类型:(add, e)或(modify, e_id, e')

3. **知识整合**
   - **经验整合**:在添加新经验前检查相似度阈值θ_sim,相似条目合并,超出容量N_max^E时删除低质量项
   - **技能整合**:将片段ΔK_i整合到全局技能文档K,超出长度L_max^K时触发精炼

**Phase II: 任务解决阶段**

面对测试任务T_test,系统采用三步动态检索和注入机制:

1. **任务分解检索**
   - 将任务分解为n_g个抽象子任务G = {g_1, ..., g_ng}
   - 每个子任务生成查询并检索相关经验
   - 公式:E_ret = ∪_{g∈G} Top-k({e∈E | cos(v_g, v_e) > τ_min})
   - 提高覆盖率,确保关键经验不被忽视

2. **经验重写**
   - MLLM_kb重写检索到的经验以适应当前任务
   - 重述条件以匹配当前任务和视觉状态
   - 用任务相关细节实例化动作
   - 丢弃明显不适用的经验

3. **技能适配**
   - 修剪不相关部分,整合重写后的经验
   - 调整代码模板为任务相关
   - 通过视觉-语义对齐确保知识根植于当前视觉上下文

执行过程中,系统记录技能和经验的使用历史H_usage,反馈到积累阶段以实现持续精炼。

### 3. 关键创新

1. **视觉引导的知识提取**:将决策与视觉证据关联,而非仅依赖文本
2. **双流互补**:技能解决低效工具使用,经验实现灵活编排
3. **跨Rollout批判**:对比分析成功/失败轨迹识别因果关系
4. **层次化整合**:减少语义冗余,保持知识库质量
5. **任务分解检索**:多方面检索提高覆盖率

## 实验设计

### 数据集

在5个基准数据集上进行评估,涵盖三个领域:

1. **视觉Agent工具使用**
   - VisualToolBench: 视觉推理与多工具操作
   - TIR-Bench: 需要精确图像分析的任务

2. **多模态搜索**
   - MMSearch-Plus: 信息检索与网页浏览
   - MMBrowseComp: 异构文本和视觉源搜索

3. **综合基准**
   - AgentVista: 超高难度,结合工具使用和复杂搜索

每个数据集随机采样100个任务作为训练集,其余用于评估。

### 工具集

根据任务类别配备灵活的工具库:
- **代码解释器**:执行Python代码进行图像处理、计算和数据分析
- **网页搜索**:使用文本查询从网络检索相关信息和链接
- **图像搜索**:执行反向图像搜索以检索与给定图像查询相关的信息和链接
- **访问**:浏览特定网页通过链接提取内容

### 基线方法

与三种最先进的学习经验基线对比:
1. **AWM (Agent Workflow Memory)**: 从历史轨迹提取可复用任务工作流
2. **DC (Dynamic CheatSheet)**: 维护问题解决策略和代码片段的演化记忆
3. **Agent-KB**: 将跨领域经验聚合成结构化知识库

### 评估指标

- **Average@4**: 4次rollout的平均成功率,反映Agent可靠性
- **Pass@4**: 至少一次成功的任务比例,反映Agent上限能力

### 实现细节

- **骨干模型**:Gemini-2.5-Pro、Gemini-3-Flash、GPT-5-mini、o4-mini
- **嵌入模型**:text-embedding-3-small
- **检索参数**:top-k = 3
- **生成参数**:temperature = 0.6, top-p = 1.0, max_turns = 20
- **跨模型迁移**:GPT-5-mini和o4-mini使用Gemini-3-Flash积累的知识

## 结果与分析

### 主要结果

XSkill在所有基准和骨干模型上都显著优于基线:

1. **相比工具基线的改进**
   - Average@4提升2.58到6.71个百分点
   - 验证了积累的经验和技能提供实质性的额外收益

2. **相比学习基线的优势**
   - 在大多数设置中超越先验方法
   - 特别是在需要复杂视觉推理和多步工具组合的任务上优势明显
   - 例如:在TIR-Bench上使用Gemini-3-Flash,达到47.75% Average@4,超越最强基线Agent-KB 11.13个百分点

3. **跨模型迁移效果**
   - GPT-5-mini和o4-mini使用迁移知识后提升2.58到4.16个百分点
   - 表明外部化知识结构在不同模型架构间保持有效

4. **指标一致性改进**
   - Average@4和Pass@4同时提升
   - 不仅提高单次rollout的可靠性,还提升了Agent的上限能力

### 消融研究

在VisualToolBench上使用Gemini-2.5-Pro进行系统消融:

**知识流贡献**
- 移除经验:Average@4下降3.04个百分点
- 移除技能:Average@4下降3.85个百分点
- 验证两个知识流都不可或缺

**两阶段架构**
- Phase 1组件(经验管理器、技能管理器)贡献更大:下降4.09和3.62个百分点
- Phase 2组件(任务分解、任务适配)贡献较小但必要:下降1.28和1.52个百分点
- 表明积累知识质量比检索机制更关键

**技能缓解低效工具使用**

错误分析显示:
- Experience Only → Skill Only:整体错误率从29.9%降至15.3%
- 语法错误:从114(20.3%)降至71(11.4%)
- 工具名称错误:从16降至2

技能提供清晰的工具模板和工作流指令,防止Agent在错误恢复上浪费推理步骤,确保有限的turn预算用于高效的问题解决。

**经验实现灵活编排**

工具使用分布分析:
- VisualToolBench:代码解释器使用从66.63%增加到76.97%
- MMSearch-Plus:代码解释器使用从6.18%增加到13.21%,图像搜索从15.43%增加到24.63%

经验捕捉特定任务的战术知识:
- 视觉推理:偏向基于代码的处理
- 多模态搜索:优先使用专门的视觉工具而非通用文本搜索

这种上下文感知的适配使XSkill超越基线的固定执行路径,实现复杂多模态任务所需的灵活工具编排。

### 进一步分析

论文还进行了开放模型评估(Qwen3-VL系列)、鲁棒性和泛化性分析等实验,结果表明:

1. **两个知识流发挥互补作用**:技能提高工具使用鲁棒性,经验实现更强的零样本跨任务迁移
2. **视觉引导的知识提取至关重要**:纯文本方法在多模态场景中不足
3. **跨模型知识迁移有效**:一个模型积累的知识可被其他模型复用

## 评价与思考

### 优势

1. **理论创新性强**
   - 首次统一任务级技能和动作级经验的双流设计
   - 视觉引导的知识提取填补了多模态Agent持续学习的空白
   - 无需参数更新即可持续改进

2. **实验设计全面**
   - 5个基准数据集,4个骨干模型,多个基线对比
   - 系统的消融研究验证各组件贡献
   - 定量和定性分析相结合

3. **性能提升显著**
   - 在所有设置上都超越基线
   - 在复杂任务上提升尤其明显(最高11.13个百分点)
   - 跨模型迁移验证了方法的通用性

4. **实用价值高**
   - 双流设计解决了Agent的两个核心瓶颈
   - 可与现有工具使用框架无缝集成
   - 支持持续精炼和迭代改进

### 局限与未来方向

1. **计算开销**
   - Phase I需要多次rollout,可能计算成本较高
   - 跨Rollout批判和知识整合需要额外的MLLM调用
   - 未来可探索更高效的积累策略

2. **知识表示**
   - Markdown和JSON格式可能不够灵活
   - 对于更复杂的任务可能需要更丰富的表示
   - 可考虑图结构或向量数据库

3. **单次积累-测试循环**
   - 实验主要展示单次循环
   - 虽然架构支持迭代精炼,但多轮循环的效果未充分验证
   - 未来可探索长期持续学习场景

4. **依赖强大的MLLM**
   - 需要能力强的MLLM_kb进行知识管理
   - 可能限制了在资源受限环境中的应用
   - 可研究轻量级知识提取方法

5. **视觉锚定的粒度**
   - 论文提到视觉引导但未详细说明如何具体锚定
   - 视觉特征的选择和表示可能影响效果
   - 需要更系统的视觉-文本对齐机制

### 关键洞察

1. **经验与技能的互补性**
   - 技能解决"如何正确使用工具"的问题
   - 经验解决"何时使用哪个工具"的问题
   - 两者结合才能实现高效且灵活的Agent

2. **视觉信息的重要性**
   - 多模态任务中视觉观察包含关键决策信号
   - 纯文本轨迹不足以捕捉完整的上下文
   - 视觉引导的知识提取和检索至关重要

3. **外部化知识的价值**
   - 将知识存储在外部库而非参数中
   - 支持跨模型迁移和持续更新
   - 无需重新训练即可改进Agent性能

4. **任务分解的必要性**
   - 复杂多模态任务往往涉及多个技术方面
   - 单一查询检索难以覆盖所有需求
   - 任务分解后多方面检索提高覆盖率

## 适用场景

XSkill框架特别适合以下场景:

### 1. 复杂多模态推理任务
- 需要结合视觉理解、工具使用和多步推理
- 例如:图像分析、视觉问答、跨模态信息检索
- 技能提供工作流指导,经验提供上下文适配

### 2. 持续部署的Agent系统
- Agent需要长期运行并不断改进
- 可从历史交互中积累经验
- 无需重新训练模型,降低部署成本

### 3. 多工具协作场景
- 任务需要组合使用多个工具
- 需要根据上下文灵活选择工具
- 例如:数据分析、网页爬取、图像处理流程

### 4. 跨领域迁移应用
- 一个领域积累的知识可迁移到相关领域
- 例如:从医学图像分析迁移到工业质检
- 减少新领域的冷启动时间

### 5. 错误恢复与鲁棒性要求高的场景
- 需要Agent能够从错误中学习
- 技能模板减少语法和工具使用错误
- 经验指导错误恢复策略

### 6. 资源受限的持续学习
- 无法承担频繁的模型重训练成本
- 需要无参数更新的学习方式
- 通过外部知识库实现能力提升

### 不太适合的场景

1. **纯文本任务**:视觉引导的优势无法体现
2. **一次性任务**:无法积累足够的经验
3. **高度专业化领域**:可能需要领域特定的知识表示
4. **实时性要求极高的场景**:知识检索和适配可能引入延迟

## 总结

XSkill提出了一个创新的双流框架,通过视觉引导的经验和技能积累,使多模态Agent能够在无需参数更新的情况下持续改进。其核心贡献在于:

1. **概念创新**:统一任务级技能和动作级经验的双流设计
2. **技术突破**:视觉引导的知识提取和跨Rollout批判机制
3. **性能验证**:在多个基准和模型上的一致性显著提升
4. **实用价值**:解决了Agent工具使用低效和编排不灵活的核心问题

这项工作为多模态Agent的持续学习开辟了新方向,证明了外部化知识结构在复杂任务中的有效性。随着多模态Agent应用场景的不断扩展,XSkill框架有望成为构建更智能、更灵活、更可靠Agent系统的重要基础。

未来研究可在以下方向深化:
- 更高效的积累策略以降低计算成本
- 更丰富的知识表示以支持更复杂任务
- 多轮迭代学习以验证长期改进效果
- 与强化学习等技术结合以优化知识提取和利用
