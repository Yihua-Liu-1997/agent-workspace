# Ranking Reasoning LLMs under Test-Time Scaling：测试时扩展下的推理模型排名研究

**论文信息**：
- 标题：Ranking Reasoning LLMs under Test-Time Scaling
- 作者：Mohsen Hariri, Michael Hinczewski, Jing Ma, Vipin Chaudhary
- arXiv ID：2603.10960
- 提交日期：2026年3月11日
- 开源库：Scorio (https://github.com/mohsenhariri/scorio)

---

## 一、背景与动机

### 测试时扩展的兴起

近年来，大语言模型（LLM）在推理任务上的表现随着测试时扩展（test-time scaling）技术的应用而显著提升。测试时扩展的核心思想是：对于同一个问题，通过多次随机采样生成多个解决方案，然后从中选择最优答案。这种方法在数学推理、代码生成等任务上取得了突破性进展。

然而，这种技术带来了一个被忽视的问题：**如何从多次随机试验中准确地评估和排名模型性能？**

### 传统排名方法的局限性

传统的 LLM 排行榜通常采用以下方式：
1. **单次运行**：每个问题只运行一次，根据正确率排名
2. **Pass@k**：运行 k 次，只要有一次正确即算通过

这些方法在测试时扩展场景下暴露出问题：
- 单次运行忽略了随机性，可能导致排名不稳定
- Pass@k 虽然考虑了多次尝试，但缺乏统计意义上的严格性
- 不同模型在不同试验次数下的相对性能可能发生变化

### 研究动机

当每个问题进行多次（如 80 次）随机试验时，我们面临的核心问题是：
- 如何从这些随机结果中估计模型的"真实"能力？
- 不同的排名方法（平均分、Bradley-Terry 模型、贝叶斯估计等）会得出不同的排序吗？
- 在试验次数有限（低预算）时，哪些方法最稳定？

本研究首次系统性地探索了这些问题，并开源了排名方法库 **Scorio**，为社区提供了统一的评估工具。

## 二、核心方法

### 2.1 问题形式化

研究将测试时扩展下的 LLM 评估形式化为一个**密集重复测量问题**：
- **M** 个问题（questions）
- **L** 个模型（models）
- **N** 次试验（trials）

每个试验产生一个二元结果：正确（1）或错误（0）。因此，评估数据形成一个三维张量 R ∈ {0,1}^{L×M×N}。

### 2.2 数据表示

论文定义了三种主要的数据表示方式：

#### 1. 点态表示（Pointwise）
- 计算每个模型-问题对的解决率
- 适用于平均准确率、IRT 模型等

#### 2. 配对表示（Pairwise）
- 统计模型之间的胜负平局次数
- 适用于 Bradley-Terry、投票规则、图方法等

#### 3. 列表/集合表示（Listwise/Setwise）
- 每次试验产生一个"获胜集合"和"失败集合"
- 适用于 Plackett-Luce 等模型

### 2.3 排名方法分类

研究比较了四大类共 40+ 种排名方法：

#### 1. 配对比较模型（Paired-Comparison Models）
- **Bradley-Terry (BT)**：经典的配对比较模型，假设每个模型有一个潜在"强度"参数
- **Rao-Kupper**：BT 模型的扩展，支持平局
- **Elo 及其变体**：动态评分系统，类似国际象棋排名
- **Glicko、TrueSkill**：带有不确定性估计的评分系统

#### 2. 项目反应理论（IRT）模型
- **Rasch 模型**：同时估计模型能力和题目难度
- **Rasch MML**：使用边际极大似然估计的贝叶斯版本
- **2PL、3PL**：包含区分度和猜测参数的扩展模型

#### 3. 投票规则（Voting Rules）
- **Borda 计数**：根据排名位置赋分
- **Condorcet 方法**：寻找能击败所有其他候选者的模型
- **Schulze 方法**：基于路径强度的 Condorcet 变体
- **Nanson、Baldwin、Copeland** 等其他投票规则
- **Majority Judgment**：基于中位数的评分方法

#### 4. 图与谱方法（Graph/Spectral Methods）
- **PageRank**：将模型排名视为网络节点重要性问题
- **Rank Centrality**：基于随机游走的排名
- **Spectral Ranking**：使用矩阵特征向量
- **HodgeRank**：处理不完整比较的谱方法
- **AlphaRank**：基于博弈论的排名方法
- **Nash equilibrium mixture**：纳什均衡混合策略

### 2.4 贝叶斯估计框架

研究提出的核心方法是**贝叶斯估计**：

#### 二元贝叶斯估计（Bayes_U@N）
对于每个模型-问题对，假设：
- 成功概率 θ ~ Beta(α, β)
- 观测 N 次试验，其中 k 次成功
- 后验均值：θ̂ = (α + k) / (α + β + N)

使用均匀先验（α = β = 1），对所有问题的后验均值求平均，得到模型分数。

#### 经验先验版本（Bayes_R0@N）
利用贪婪解码（greedy decoding）作为辅助信号：
- 对每个问题进行一次贪婪解码（温度=0）
- 将贪婪结果作为先验的一部分
- 这相当于向先验中添加 O(1) 个伪计数

#### 区间估计与保守排名
- **可信区间**：使用后验分位数（如 5% 分位数）进行保守排名
- **置信区间**：通过 bootstrap 重采样估计
- **成对优越概率**：计算 Pr(θ_i > θ_j | R)

### 2.5 金标准的定义

研究明确定义了"真实排名"的**金标准**：**Bayes_U@80**
- 使用所有 80 次试验
- 均匀贝叶斯先验
- 这是基于边际成功概率的估计，具有明确的操作意义

研究证明了一个重要结论：**不同排名方法在理论上可能收敛到不同的极限排序**，因此必须明确定义金标准。论文通过一个最小反例（8 个问题，3 个模型，1 次试验）证明：平均分排名和 BT 排名即使在无限预算下也可能不一致。

## 三、实验设计

### 3.1 数据集

研究使用了四个数学奥林匹克级别的基准测试：
1. **AIME'24**：2024 年美国数学邀请赛（30 题，平均准确率 62.0%）
2. **AIME'25**：2025 年美国数学邀请赛（30 题，平均准确率 53.3%）
3. **HMMT'25**：2025 年哈佛-麻省理工数学锦标赛（30 题，平均准确率 33.3%，最困难）
4. **BrUMO'25**：2025 年布朗大学数学奥林匹克（30 题，平均准确率 58.8%）

### 3.2 模型

评估了 **20 个推理模型**，包括：
- **商业模型**：OpenAI o1、DeepSeek R1、Qwen QwQ 等
- **开源模型**：多种规模的推理模型
- 所有模型使用统一的采样参数

### 3.3 试验设计

每个问题进行 **80 次随机试验**：
- 使用 top-p 采样
- 记录每次试验的正确性
- 同时对每个问题进行一次贪婪解码（用于经验先验）

### 3.4 评估指标

使用 **Kendall's τ_b** 作为排名一致性的度量：
- τ = 1：完全一致的排序
- τ = 0：无关
- τ = -1：完全相反

研究测量两种一致性：
1. **金标准一致性**：与 Bayes_U@80 的相关性
2. **自一致性**：与自身所有试验排名的相关性（method@80）

### 3.5 低预算协议

为了研究低预算场景，研究采用以下协议：
- 从 80 次试验中随机采样 N 次（N = 1, 2, 5, 10, 20, 40）
- 重复采样 80 次，计算 τ_b 的均值和标准差
- 比较不同排名方法的稳定性和准确性

## 四、结果与分析

### 4.1 高预算（N=80）结果

**主要发现**：在拥有充足试验次数时，大多数合理的排名方法产生几乎相同的排序。

具体数据：
- 四个数据集上，不同方法与金标准 Bayes_U@80 的平均 τ_b 在 **0.93–0.95** 之间
- **19–34 种方法**恢复出完全相同的排序
- 这意味着在高预算下，排名方法的选择相对不敏感

**表现最好的方法**：
- 贝叶斯估计（Bayes_U@80）
- 简单平均分（avg@80）
- Borda、Copeland、Schulze 等投票规则
- PageRank、Spectral Ranking 等图方法

**表现较差的方法**：
- Bradley-Terry ML（在 N=1 时最差，τ_b ≈ 0.81）
- Pass@2（在 HMMT'25 上 τ_b = 0.937）
- Minimax 某些变体

### 4.2 低预算（N=1）结果

**关键发现**：单次试验时，排名方法的差异显著放大。

**金标准一致性排名**（Combined 基准，N=1）：
1. **第一梯队**（τ_b ≈ 0.865）：Bayes、Borda、Copeland、PageRank、Spectral、Schulze 等 20+ 种方法
2. **Bradley-Terry ML**：τ_b ≈ 0.806（倒数第二）
3. **Bayes_greedy**：τ_b ≈ 0.786（最差）

**自一致性排名**：
1. Nanson avg ties：τ_self = 0.892
2. Rasch MML credible：τ_self = 0.883
3. 大多数方法在 0.86 左右

**重要观察**：**高自一致性不等于高金标准一致性**
- Nanson 方法自一致性最高，但金标准一致性中等
- BT-ML 自一致性不错，但金标准一致性最差
- 这说明某些方法虽然稳定，但可能收敛到"错误"的排序

### 4.3 经验先验的效果

**方差减少**：使用贪婪解码作为经验先验（Bayes_R0@1），在 N=1 时显著降低方差：
- AIME'24：标准差减少 42%
- BrUMO'25：标准差减少 52%（最大）
- HMMT'25：标准差减少 16%（最小）

**均值偏移的双刃剑**：
- **正面案例**：BrUMO'25，Δτ = +0.049（提升金标准一致性）
- **负面案例**：HMMT'25，Δτ = -0.022（降低金标准一致性）

**贪婪-采样对齐性**（Greedy-Sampling Alignment）：
研究定义了 τ_G-S：贪婪解码排名与 80 次采样排名的相关性。
- 当 τ_G-S 高时（如 BrUMO'25: 0.768），经验先验有帮助
- 当 τ_G-S 低时（如 HMMT'25: 0.635），经验先验可能有害

**解释**：经验先验相当于向贪婪排序的收缩估计器。当贪婪解码能代表采样行为时，它提供有用信息；当两者分歧时（例如贪婪在难题上探索不足），它引入系统性偏差。

### 4.4 Bootstrap 模型池稳健性

研究通过 bootstrap 采样 5、10、15 个模型子集，测试结论的稳健性：

**发现**：
- 简单/中等难度的基准（AIME'24、AIME'25、BrUMO'25）：Bayes_R0@N 始终最优
- 困难基准（HMMT'25）和组合基准：29-30 种方法并列最优
- **更大的模型池主要减少跨子集方差**，而不改变定性结论
- 从 5 个到 15 个模型，标准差从 0.209 降至 0.057（AIME'24）

### 4.5 分类排名（Categorical Ranking）

研究将框架扩展到多类别输出，设计了 25 种分类方案：

**基础信号**（每个完成提取 9 个特征）：
- 格式（has_box）：是否有 boxed 答案
- 正确性（is_correct）
- 生成长度（token_ratio）
- 停止原因（repeated_pattern）
- 解码置信度（prompt_bpt, completion_bpt）
- 外部验证器（CompassVerifier 的 3 个输出）

**8 种代表性方案**：
1. **Conservative**：惩罚高置信度的错误
2. **Efficiency-adjusted**：对冗长的正确答案降权
3. **Format-aware**：奖励 boxed 的正确答案
4. **Balanced composite**：综合格式和置信度
5. **OOD-robust**：奖励分布内的答案
6. **Rare-event**：包含弃权选项
7. **Verifier-calibrated**：惩罚验证器的假阳性
8. **Verifier-only**：完全依赖外部验证器，无真实标签

**结果**（Combined 基准，N=1，11 个模型）：
- **Conservative** 方案金标准一致性最高（τ_GS = 0.856）
- **Verifier-only** 自一致性最高（τ_self = 0.897），但金标准一致性最低（τ_GS = 0.824）
- **负相关**：τ_GS 与 τ_self 在不同方案间呈负相关

**关键洞察**：使用更多辅助信号（如验证器、置信度）可以提高单次试验的稳定性，但可能系统性偏离正确性排序。

## 五、评价与思考

### 5.1 研究贡献

#### 理论贡献
1. **首次形式化**测试时扩展下的密集基准排名问题
2. **证明**不同排名方法在理论上可能收敛到不同极限（附录 C 的最小反例）
3. **明确定义**金标准（Bayes_U@N），为评估提供统一参考

#### 实践贡献
1. **Scorio 开源库**：实现了 40+ 种排名方法，为社区提供统一工具
2. **系统性比较**：在 4 个基准、20 个模型、80 次试验的规模上比较方法
3. **实用建议**：针对不同预算场景给出方法选择指南

#### 方法论贡献
1. 提出用 **Kendall's τ_b** 作为排名一致性的标准度量
2. 区分**金标准一致性**和**自一致性**两个维度
3. 引入**贪婪-采样对齐性**作为经验先验有效性的诊断指标

### 5.2 局限性

#### 数据范围
- 仅评估数学推理任务
- 未涉及开放生成、代码、多模态等其他任务
- 题目数量相对较少（每个基准 30 题）

#### 评估粒度
- 仅使用二元正确性，未考虑部分得分
- 未探索 rubric-based 评分
- 分类排名的方案设计有一定任意性

#### 先验来源
- 经验先验仅来自贪婪解码
- 未探索其他先验来源（如小模型评估、人类先验等）
- 先验的潜在偏差需要更深入分析

### 5.3 深层思考

#### 1. 排名的本质是什么？

这篇论文揭示了一个深刻问题：**在随机性下，"真实排名"并非唯一确定**。

- 平均分方法目标是边际成功概率 p_l
- Bradley-Terry 目标是解释配对胜负率的潜在强度
- 这两者是同一联合分布的不同摘要

因此，"哪个模型更好"取决于我们关心什么：
- 如果关心"平均能解决多少问题"→ 用平均分
- 如果关心"在直接对决中谁更强"→ 用 BT
- 如果有领域知识（如题目难度）→ 用 IRT

#### 2. 为什么高预算下方法趋于一致？

研究发现在 N=80 时，19-34 种方法给出完全相同的排序。这暗示：
- 当数据充足时，不同统计目标指向相似的排序
- 排名的"真实"结构在大量数据下变得清晰
- 方法的差异主要体现在如何处理噪声和不确定性

#### 3. 经验先验的哲学

Bayes_R0@N 的效果依赖于贪婪-采样对齐性，这引出一个问题：
- **我们应该评估什么？**
  - 贪婪解码的行为？（确定性、可复现）
  - 采样解码的行为？（探索性、代表性）
  - 还是两者的某种综合？

论文倾向于后者，但这也意味着：
- 评估本身变成了一个建模选择
- 没有完全"中立"的评估方法

#### 4. 自一致性的陷阱

研究明确指出：**高自一致性不等于正确性**。这是一个重要警告：
- 某些方法可能稳定地收敛到错误答案
- 在缺乏金标准时，自一致性可能误导
- 这对实际应用有重要启示：在小样本验证时，不能只看稳定性

#### 5. 分类排名的前景

分类排名的实验显示了一个有趣的权衡：
- 使用更多信号 → 更稳定，但可能偏离目标
- 仅用正确性 → 更准确，但噪声更大

这暗示未来的方向可能是：
- 自适应方案：根据问题特征选择评估策略
- 多目标优化：同时考虑正确性、置信度、效率
- 元学习：从历史数据学习最优评估方案

### 5.4 与现有工作的关系

#### 与 Pass@k 的对比
- Pass@k 是阈值化的，丢失了梯度信息
- 贝叶斯估计保留了不确定性
- 论文显示贝叶斯方法在低预算时更稳定

#### 与 Chatbot Arena 的关系
- Chatbot Arena 使用人类偏好，是稀疏的配对比较
- 本研究是密集的重复测量，更接近传统心理测量
- 两者的统计方法有交叉（BT、Elo），但应用场景不同

#### 与 IRT 在 NLP 中的应用
- 近期工作将 IRT 引入 LLM 评估
- 本研究扩展到重复测量，连接了 IRT 与经典测试理论
- Rasch 模型的表现不错，但未显著优于简单平均

## 六、适用场景

### 6.1 方法选择指南

#### 高预算场景（N ≥ 40）
- **推荐**：Bayes_U@N 或简单平均（avg@N）
- **理由**：几乎所有合理方法都给出相似排序
- **注意**：避免过于复杂的方法（如 BT-ML），它们可能在低数据时不稳定

#### 中等预算（10 ≤ N < 40）
- **推荐**：Bayes_U@N，Borda，Copeland，PageRank
- **理由**：这些方法在 N=10-20 时已接近收敛
- **可考虑**：Rasch MML（如果题目难度信息有用）

#### 低预算（1 ≤ N < 10）
- **谨慎推荐**：Bayes_R0@N（经验先验版本）
- **前提条件**：
  1. 先进行小规模 pilot 研究，计算 τ_G-S
  2. 如果 τ_G-S > 0.7，经验先验可能有用
  3. 如果 τ_G-S < 0.65，避免使用经验先验
- **安全选择**：Bayes_U@N（均匀先验）
- **避免**：Bradley-Terry ML（在 N=1 时表现最差）

#### 单次试验（N = 1）
- **最佳**：Bayes（均匀先验）、Borda、Copeland、PageRank、Spectral
- **考虑**：Nanson avg ties（最高自一致性，但金标准一致性中等）
- **避免**：BT-ML、Minimax 某些变体、Bayes_greedy（在 HMMT'25 上）

### 6.2 分类排名的适用场景

#### 推荐使用分类排名的情况
1. **无真实标签**：Verifier-only 方案可以在缺乏 ground truth 时使用
2. **成本敏感**：Efficiency-adjusted 方案可以惩罚冗长输出
3. **高价值决策**：Conservative 方案惩罚高置信度错误，适合医疗、法律等场景
4. **分布外检测**：OOD-robust 方案可以识别异常问题

#### 不推荐的情况
1. **纯正确性评估**：简单二元评估已足够
2. **小样本**：分类方案增加复杂性，可能引入更多噪声
3. **信号质量未知**：如果验证器等辅助信号可靠性未经验证，可能误导

### 6.3 实际部署建议

#### 基准测试设计
1. **预试验**：在正式评估前，用小规模 pilot 数据检查：
   - 贪婪-采样对齐性 τ_G-S
   - 不同方法的稳定性
   - 选择合适的 N（试验次数）
2. **预算分配**：
   - 如果计算资源充足，N ≥ 40，用简单方法即可
   - 如果资源紧张，N = 1-5，使用经验先验（如果对齐性高）
3. **模型池大小**：
   - 至少 10 个模型，以减少随机性
   - 模型池越多样，排名越有意义

#### 排行榜构建
1. **透明性**：
   - 明确报告使用的排名方法
   - 报告试验次数 N
   - 如果使用经验先验，报告其来源
2. **不确定性量化**：
   - 报告 τ_b 的置信区间
   - 对于关键排名差异，进行显著性检验
3. **多维度展示**：
   - 同时报告金标准一致性和自一致性
   - 展示不同方法下的排名（如果存在分歧）

#### 模型选择（从业者视角）
1. **如果只看排行榜**：
   - 优先选择使用 Bayes_U@N 或 avg@N 的排行榜
   - 对 N < 10 的排行榜保持怀疑
   - 注意经验先验可能引入的偏差
2. **如果自己评估**：
   - 使用 Scorio 库
   - 至少运行 N = 5-10 次试验
   - 比较多种排名方法，看是否一致
3. **如果资源极度有限**：
   - N = 1 + 贪婪解码
   - 使用 Bayes_R0@1，但验证 τ_G-S
   - 承认评估的不确定性

### 6.4 未来应用方向

#### 1. 自适应测试
- 根据 model performance 动态调整试验次数
- 对接近的模型对增加采样，对明显优劣的减少

#### 2. 多任务扩展
- 将框架扩展到代码、推理、知识问答等多任务
- 研究跨任务排名的聚合方法

#### 3. 在线评估
- 类似 Chatbot Arena，但基于自动基准而非人类偏好
- 实时更新排名，处理模型更新

#### 4. 成本-效益优化
- 给定预算，最优分配试验次数和问题数量
- 研究问题选择策略（主动学习）

## 总结

这篇论文是测试时扩展时代 LLM 评估的里程碑式工作。它不仅提供了理论框架和实用工具（Scorio），更重要的是揭示了一个深刻事实：**在随机性下，排名本身就是一个需要谨慎定义和估计的统计目标**。

核心启示：
1. **高预算时**：方法选择相对不重要，简单方法（Bayes_U@N 或 avg@N）即可
2. **低预算时**：方法选择至关重要，经验先验是双刃剑
3. **始终验证**：自一致性不等于正确性，需要明确金标准
4. **透明报告**：评估方法本身是建模选择，必须明确说明

对于研究者和从业者，这篇论文提供了清晰的行动指南：使用 Scorio 库，根据预算选择方法，验证经验先验的对齐性，透明报告评估细节。在 LLM 能力快速发展的今天，建立严谨、可复现的评估体系比以往任何时候都更重要。
