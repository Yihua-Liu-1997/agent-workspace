# RoboClaw：可扩展长时域机器人任务的智能体框架

**论文链接**：https://arxiv.org/abs/2603.11558  
**发布时间**：2026年3月  
**机构**：AgiBot、新加坡国立大学、上海交通大学  

---

## 一、背景与动机

### 现实挑战

随着视觉-语言-动作（VLA）系统的发展，基于语言指令的机器人操作展现出了巨大潜力。然而，将这些系统扩展到**长时域任务**（Long-Horizon Tasks）仍然面临关键挑战：

1. **人工成本高昂**：传统的机器人数据收集流程需要大量人工参与——操作员需要收集演示、反复重置环境、监控失败、过滤轨迹、评估模型性能等。随着任务复杂度的增加，这种以人为中心的数据收集和部署过程变得越来越昂贵且难以扩展。

2. **流程割裂**：数据收集、模型学习和任务执行通常由不同的流程处理，导致系统管道中出现信息鸿沟。任务状态、子任务边界或成功标准的解释在不同阶段可能存在差异，难以在整个系统中保持一致的任务语义。

3. **训练与部署不匹配**：当数据收集、模型学习和任务执行由独立过程驱动时，训练数据覆盖的状态分布往往无法反映部署时遇到的条件，导致训练和执行之间的不匹配。

4. **长时域任务脆弱**：在语义和分布上的不一致使得长时域任务特别脆弱，小错误可能在执行过程中传播和级联。

### 核心洞察

研究团队认识到，关键挑战在于如何为可扩展的语言驱动机器人系统建立**统一的语义表示和决策机制**，跨越数据收集、策略学习和执行三个阶段。

## 二、核心方法

RoboClaw 提出了一个统一的智能体架构，包含三大核心创新：

### 1. 纠缠动作对（Entangled Action Pairs, EAP）

这是 RoboClaw 最具创新性的机制之一：

**设计理念**：为每个操作策略配对两个行为：
- **正向执行策略**（Forward Execution Policy）：执行实际的操作任务
- **逆向恢复策略**（Inverse Reset Policy）：将环境恢复到初始状态

**工作原理**：
```
初始状态 → 正向操作 → 目标状态 → 逆向恢复 → 初始状态 → ...
```

这种设计形成了**自重置循环**（Self-Resetting Loop），使机器人能够：
- 在无需人工干预的情况下持续收集数据
- 自动返回到可重用的前置条件区域
- 实现在线数据收集和策略迭代优化

**关键优势**：逆向策略被故意设计得比正向任务更简单，因此成功率更高（实验显示达到 72%-86%），从而支持稳定的自动数据收集循环。

### 2. VLM 驱动的元控制器

RoboClaw 使用现成的视觉-语言模型（VLM）作为高层控制器：

**核心能力**：
- **上下文学习**（In-Context Learning）：通过少样本示例快速适应新任务
- **链式思考推理**（Chain-of-Thought）：对环境观察和结构化记忆进行推理
- **动态决策**：根据当前上下文选择下一个子任务或恢复策略

**结构化记忆系统**：
- **角色身份**（Role Identity）：定义当前操作模式和可用工具
- **任务级记忆**（Task-Level Memory）：记录全局任务、分解的子任务及其执行状态
- **工作记忆**（Working Memory）：存储短期执行上下文，如当前激活的技能和工具调用历史

### 3. 部署时过程监督与技能调度

在任务执行阶段，RoboClaw 不仅仅是执行预定义的计划：

**监控机制**：
- 定期查询环境摘要和机器人状态
- 评估子任务是否成功完成
- 检测重复失败或意外环境状态

**自适应策略**：
- **重试**：对于非退化失败（如抓取偏移），直接重试相同策略
- **恢复**：对于退化失败（如瓶子倒下），调用恢复策略
- **重新规划**：选择替代技能
- **人工干预**：当自主恢复失败或触发安全条件时，请求人工介入

**生命周期学习**：部署期间生成的轨迹被记录并整合到训练数据集中，形成闭环学习系统。

### 4. MCP 工具接口

RoboClaw 通过模型上下文协议（Model Context Protocol）提供统一的工具接口：

**核心工具**：
- Start Policy：启动策略
- Terminate Policy：终止策略
- Change Policy：切换策略
- Env Summary：查询环境摘要
- Fetch Robot Stats：获取机器人状态
- Call Human：请求人工干预

这种设计使得高层推理能够无缝转化为可执行动作。

## 三、实验设计

### 实验平台

**硬件**：Agibot G01 双臂移动操作机器人
- 移动底座安装
- 20个自由度（不包括末端执行器）
- 每个手臂配备 AGIBOT OmniPicker 自适应夹持器

### 评估场景

在四个真实世界场景中评估：
1. **卧室梳妆台**：整理物品
2. **厨房货架**：排列物体
3. **书房桌面**：整理物品
4. **便利店货架**：按指令选择产品

### 四个关键问题

1. RoboClaw 能否显著提高数据收集效率？
2. RoboClaw 能否提高子任务策略的成功率？
3. RoboClaw 能否更好地处理长时域任务？
4. RoboClaw 能否从失败中学习？

### 四个子任务

1. **身体乳放置**（Body Lotion）：长距离抓取和放置，相机视角变化大
2. **妆前乳放置**（Primer）：放入抽屉并关闭，需要精确放置和留出关闭空间
3. **口红插入**（Lipstick）：插入窄槽，需要精确对齐
4. **纸巾擦拭**（Tissue Wipe）：擦拭桌面液体，需要持续表面接触

## 四、结果与分析

### 1. 数据收集效率

**人类时间投入**：
- 传统方法需要 **2.16倍** 更多人类时间
- 人工干预次数：传统方法需要 **8.04倍** 更多干预

**解读**：RoboClaw 的 EAP 机制使得机器人能够自主监控环境状态并重复执行纠缠动作对，持续生成新轨迹，大幅降低了对人工演示的依赖。

### 2. 子任务策略改进

**逆向策略成功率**（50次试验）：
- 身体乳：36/50（72%）
- 妆前乳：38/50（76%）
- 口红：43/50（86%）
- 纸巾擦拭：39/50（78%）

**正向策略迭代改进**（从第1到第5次迭代）：
- 身体乳：21/50 → 43/50（+104%）
- 妆前乳：23/50 → 40/50（+74%）
- 口红：2/50 → 23/50（+1050%）
- 纸巾擦拭：11/50 → 26/50（+136%）

**关键洞察**：
- 闭循环收集的轨迹提供了更有信息量的训练数据
- 在相同人工演示预算下提高了策略鲁棒性
- 逆向和正向策略的不对称性有利于 EAP 数据收集机制

### 3. 长时域任务表现

**实验设置**：梳妆台整理任务，包含四个子任务

**结果**：
- RoboClaw 在长时域任务上的成功率比基准方法高 **25%**
- 显著优于端到端 VLA 基准和独立子任务成功率的乘积

**原因分析**：
- RoboClaw 能够监控任务进度
- 自动调用恢复策略
- 在失败时重新规划

### 4. 从失败中学习

**失败分类**：

1. **非退化失败**（Non-degrading Failures）：
   - 环境状态基本不变
   - 可通过重试相同策略解决
   - 例如：抓取偏移导致空抓

2. **退化失败**（Degrading Failures）：
   - 失败改变了环境状态
   - 需要额外的恢复动作
   - 例如：抓取失败导致瓶子倒下

**学习机制**：
- 早期需要人工干预来恢复场景
- 随着经验积累，恢复行为被整合到策略库中
- 后期可自主调用恢复策略

## 五、评价与思考

### 优势

1. **统一框架**：首次将数据收集、策略学习和任务执行统一在单个 VLM 驱动的智能体循环中，确保了整个系统生命周期中一致的任务语义和决策逻辑。

2. **显著降低人工成本**：人类时间投入减少 53.7%，人工干预减少 8.04 倍，这对实际部署具有重要意义。

3. **自适应能力强**：能够处理非退化和退化失败，支持动态重规划和恢复。

4. **生命周期学习**：部署不仅是执行任务，还是持续改进技能库的经验来源。

5. **模块化设计**：技能、工具和策略的分层抽象使得系统易于扩展和维护。

### 局限性

1. **延迟问题**：依赖云端大模型可能引入延迟，影响实时性能。

2. **逆向策略假设**：假设存在实用的逆向重置行为来构建可重用的环境状态，但这并不总是可行。

3. **任务复杂度限制**：实验任务相对简单（四个子任务），更复杂的长时域任务（数十个子任务）的性能尚未验证。

4. **安全考虑**：虽然有人工干预机制，但自主操作的安全性仍需进一步验证。

### 与相关工作对比

**vs. 传统遥操作**（AnyTeleop, GELLO, Mobile ALOHA）：
- 传统方法需要大量人工演示和环境重置
- RoboClaw 实现了自主数据收集

**vs. 半自动化系统**（RoboCopilot, Genie Centurion, VLAC）：
- 这些系统仍需要人工干预或预定义场
- RoboClaw 提供完全学习驱动的自动化数据收集

**vs. 模拟数据生成**（MimicGen, RoboCasa, HumanoidGen）：
- 模拟环境存在模拟到真实的差距
- RoboClaw 直接在真实世界收集数据

**vs. 现有 VLA 系统**（RT-2, OpenVLA, π₀）：
- 这些系统在长时域任务中容易出现错误累积
- RoboClaw 提供运行时监督和恢复机制

## 六、适用场景

### 最适合的应用场景

1. **家庭服务机器人**：
   - 整理房间、收拾物品
   - 需要频繁重复类似任务
   - 环境相对结构化

2. **仓储物流**：
   - 货物分拣和放置
   - 需要长时间自主操作
   - 任务可分解为多个子任务

3. **零售商店**：
   - 货架整理和补货
   - 需要处理多种商品
   - 操作环境相对固定

4. **实验室自动化**：
   - 重复性实验操作
   - 需要精确执行多步骤流程
   - 可从失败中学习和改进

### 不太适合的场景

1. **高度非结构化环境**：如户外、建筑工地等，难以设计可靠的逆向策略。

2. **单次执行任务**：不需要重复执行的任务，EAP 的优势无法体现。

3. **安全关键应用**：医疗手术、危险品处理等，需要更严格的安全保证。

4. **极端复杂任务**：涉及数十个子任务的复杂操作，当前框架的可扩展性尚未验证。

### 部署建议

1. **初始阶段**：从简单任务开始，积累少量人工演示
2. **迭代优化**：通过 EAP 机制自主收集数据，逐步提高策略性能
3. **监控与干预**：保持人工监督机制，确保安全
4. **持续学习**：利用部署数据持续改进技能库

---

## 七、总结

RoboClaw 代表了机器人学习系统从"人工驱动"向"智能体驱动"的重要转变。通过统一数据收集、策略学习和任务执行的框架，并引入创新的 EAP 机制，该系统在显著降低人工成本的同时，提高了长时域任务的成功率。

虽然仍存在延迟、安全性和可扩展性等挑战，但随着 VLM 和 VLA 模型的持续改进，RoboClaw 提供了一个有前景的基础，可以自然地集成更强的模型，扩展到更广泛的机器人能力（如导航、移动操作、多模态交互），并集成更丰富的感知、规划和执行工具，支持更自主和适应性的机器人系统。

这项工作为可扩展的具身 AI 系统开辟了新的研究方向，展示了统一智能体框架在解决长时域机器人操作挑战方面的巨大潜力。
