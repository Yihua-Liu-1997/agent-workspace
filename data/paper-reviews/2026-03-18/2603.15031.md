# Attention Residuals：用注意力机制重新定义残差连接

> **论文**: arXiv:2603.15031 | **作者**: Kimi Team (Moonshot AI) | **日期**: 2026-03-16
> **代码**: https://github.com/MoonshotAI/Attention-Residuals

---

## 背景与动机

残差连接（Residual Connection）是现代大语言模型的基石。自从 ResNet 提出以来，`h_l = h_{l-1} + f_{l-1}(h_{l-1})` 这个简单的加法公式几乎出现在每一个 Transformer 架构中。我们通常把它理解为一条"梯度高速公路"——恒等映射让梯度可以绕过中间变换，直接回传到浅层，使得深度网络的训练成为可能。

但 Kimi 团队在这篇工作中指出了一个长期被忽视的问题：残差连接不仅影响梯度流动，它还决定了**信息如何在深度维度上聚合**。展开递推公式可以看到，每一层的输入实际上是对前面所有层输出的等权求和——权重全部是固定的 1。这种"一刀切"的聚合方式带来了三个根本性缺陷：

1. **PreNorm 稀释效应**：在 PreNorm 架构中，隐藏状态的范数随深度以 O(L) 的速度增长。每一层对最终输出的相对贡献被持续稀释，深层为了保持影响力不得不产生越来越大的输出，可能导致训练不稳定。
2. **无法选择性访问**：不同类型的层（如 attention 和 MLP）对信息的需求不同，但它们只能接收到同一个压缩后的混合状态，无法选择性地关注特定层。
3. **不可逆的信息损失**：一旦信息在加法聚合中被"平均掉"，后续层无法恢复。

作者提出了一个精彩的类比：**深度维度的残差连接，与序列维度的 RNN 递推在数学上是对偶的**。就像 Transformer 用自注意力替代了 RNN 的序列递推一样，为什么不能用注意力机制来替代残差连接的深度递推？这个洞察直接催生了 Attention Residuals。

## 核心方法

### Full Attention Residuals

AttnRes 的核心思想极其优雅：将固定权重的求和替换为 softmax 注意力。具体而言，第 l 层的输入变为：

```
h_l = Σ_{i=0}^{l-1} α_{i→l} · v_i
```

其中 `v_0 = h_1`（token embedding），`v_i = f_i(h_i)`（第 i 层的输出），注意力权重通过一个**可学习的伪查询向量** `w_l ∈ R^d` 计算：

```
α_{i→l} = φ(w_l, k_i) / Σ_j φ(w_l, k_j)
φ(q, k) = exp(q^T · RMSNorm(k))
```

关键设计点：查询 `w_l` 是一个与输入无关的可学习参数，而不是从隐藏状态投影得到的。这个看似简单的选择带来了巨大的工程优势——同一 block 内所有层的伪查询可以预先知道，允许批量并行计算。RMSNorm 的加入防止了输出范数较大的层主导注意力分布。

### Block Attention Residuals

Full AttnRes 在小规模训练中没有额外内存开销（前向激活本来就要为反向传播保留），但在大规模分布式训练中，激活重计算和流水线并行意味着所有层的输出必须被显式保存和跨 stage 传输，内存和通信开销为 O(Ld)。

为此作者提出了 Block AttnRes：将 L 层分成 N 个 block，block 内用标准残差求和压缩为单个表示 `b_n = Σ_{j∈B_n} f_j(h_j)`，block 间用注意力聚合这 N 个 block 表示。这把内存和通信从 O(Ld) 降到了 O(Nd)。实验表明 N≈8 即可恢复大部分收益。

### 系统优化

论文花了不少篇幅讨论工程实现，这也是它能落地的关键：

- **跨 stage 缓存**：流水线并行中，利用同一物理设备处理多个虚拟 stage 的特性缓存 block 表示，通信量降低 V 倍（V 为虚拟 stage 数）。
- **两阶段推理**：Phase 1 批量计算 block 间注意力，Phase 2 顺序处理 block 内依赖，通过 online softmax 合并结果。每层 I/O 从 O(Ld) 降至 O((S+N)d)。
- **长上下文预填充**：将 block 表示沿序列维度分片到 P 个张量并行设备上，128K 序列的额外内存从 15GB 降至每设备 ~1.9GB，配合 chunked prefill 可进一步降至 <0.3GB。

训练开销 <4%（流水线场景），推理延迟开销 <2%。

## 实验设计

### Scaling Laws

在 5 个模型规模（194M-528M 激活参数，MoE 架构）上拟合 scaling law 曲线，三种配置（Baseline、Full AttnRes、Block AttnRes N=8）共享超参数，确保公平对比。

### 主实验

将 Block AttnRes 集成到 Kimi Linear 架构中（48B 总参数 / 3B 激活参数，27 个 Transformer block，8/256 路由专家 + 1 共享专家），在 1.4T token 上预训练。采用 Muon 优化器 + WSD 学习率调度，全局 batch size 8M token，上下文长度 4096→32K 渐进扩展。

### 消融实验

在 436M 参数（16 层）模型上进行组件消融：对比 DenseFormer、mHC、不同注意力形式（softmax vs sigmoid）、多头 vs 单头、RMSNorm 的重要性、block size 扫描等。

### 架构搜索

在固定计算预算（~6.5×10^19 FLOPs）和参数量（~2.3×10^8）下，扫描 25 种 d_model/L_b × H/L_b 配置。

## 结果与分析

### Scaling Laws

三种变体的 scaling 指数几乎相同（约 -0.057~-0.058），但 AttnRes 一致性更低：

- Baseline: L = 1.891 × C^{-0.057}
- Block AttnRes: L = 1.870 × C^{-0.058}
- Full AttnRes: L = 1.865 × C^{-0.057}

在 5.6 PFLOP/s-days 处，Block AttnRes 达到 1.692 vs Baseline 1.714，**等效于 1.25 倍计算优势**。同时，Block AttnRes 的每层内存 I/O 仅 5.5d，远低于 mHC 的 34d。

### 训练动态

训练过程中的可视化揭示了深刻的现象：

- **输出幅度**：Baseline 的隐藏状态范数随深度单调增长（PreNorm 稀释），而 Block AttnRes 将增长限制在每个 block 内，形成有界的周期性模式。
- **梯度幅度**：Baseline 的梯度集中在最早的几层，而 AttnRes 的 softmax 权重引入了层间竞争，使梯度分布显著更均匀。

### 下游任务

在所有 15 个基准测试上一致提升，尤其突出的是：

| 基准 | Baseline | AttnRes | 提升 |
|------|----------|---------|------|
| GPQA-Diamond | 36.9 | **44.4** | +7.5 |
| Math | 53.5 | **57.1** | +3.6 |
| HumanEval | 59.1 | **62.2** | +3.1 |
| C-Eval | 79.6 | **82.5** | +2.9 |
| MMLU | 73.5 | **74.6** | +1.1 |

多步推理和代码生成任务提升最大，这符合直觉——**更好的深度信息流有利于组合性任务**，深层可以选择性地检索和组合浅层表征。

### 消融发现

- DenseFormer（固定标量权重）与 Baseline 几乎无差别（1.767 vs 1.766），说明**输入依赖的动态权重至关重要**。
- Softmax 优于 sigmoid（1.737 vs 1.741），竞争性归一化迫使更锐利的选择。
- 多头注意力反而有害（1.752 vs 1.746）——最优的深度混合在通道维度上几乎是均匀的。
- 去掉 RMSNorm 导致 Full AttnRes 从 1.737 退化为 1.743，Block 从 1.746 退化为 1.750。
- 伪查询必须初始化为零，否则训练初期会剧烈震荡。

### 架构偏好

AttnRes 将最优架构的 d_model/L_b 从 ~60 移向 ~45，即在固定参数预算下**更偏好更深更窄的网络**。在 25 种配置中每种都一致优于 Baseline 0.019-0.063。

### 注意力模式可视化

学到的深度注意力权重呈现出清晰的规律：对角占优（局部性仍然是主要信息通路）、embedding 始终保持非零权重、pre-attention 层保持更广的感受野而 pre-MLP 层更依赖近邻。Block AttnRes 保留了这些结构特征，同时表现出更锐利的决策性权重分布。

## 评价与思考

**这是一篇令人兴奋的工作。** 它的核心洞察——残差连接与 RNN 递推的深度-序列对偶性——简洁而深刻，用 attention 替代固定聚合的方式既自然又有理论支撑。统一的结构化矩阵分析框架（将残差变体视为深度维度的线性/softmax 注意力）为整个领域提供了新的理解工具。

**优点**：
- **思想优雅，落地扎实**：从一个清晰的洞察出发，推导出方法、工程方案、实验验证，形成完整闭环。不是那种"加个模块刷点数"的工作。
- **工程完备性**：跨 stage 缓存、两阶段推理、内存高效预填充——这些优化使得方法在实际大规模训练中开销极小（<4% 训练，<2% 推理），这是很多架构创新论文做不到的。
- **分析深入**：训练动态可视化、注意力模式分析、结构化矩阵视角的讨论，都超出了单纯的性能报告。

**局限与疑问**：
- 每层增加一个 RMSNorm 和一个 d 维伪查询向量虽然参数量可忽略，但在极大规模（数百B参数）下对内存的实际影响值得进一步量化。
- Block AttnRes 的 block 数量 N≈8 是经验选择，文中虽做了 block size 扫描但未给出严格的理论指导。
- 论文没有与 DenseFormer 和 mHC 在 48B 规模上进行直接对比，小模型的消融结论是否完全外推有待验证。
- 伪查询与输入无关（input-independent）虽然降低了开销，但消融显示 input-dependent query 可以进一步降低 loss（1.731 vs 1.737），是否存在一种轻量的折中方案？

**理论意义**：论文通过结构化矩阵分析揭示了现有残差变体本质上是深度维度的**线性注意力**，而 AttnRes 是深度维度的 **softmax 注意力**。这个"线性→softmax"的跃迁与序列维度上从 RNN 到 Transformer 的跃迁完全对应，是一个非常有启发性的统一框架。

## 适用场景

- **需要深度信息流优化的任何 Transformer 模型**，特别是 MoE 架构（论文基于 Kimi Linear 验证）。
- **组合性推理任务**（多步数学推理、代码生成），这些任务受益于深层对浅层表征的选择性访问。
- **大模型预训练**：作为标准残差连接的 drop-in 替换，几乎零成本获得 ~1.25× 等效计算优势。
- **深层网络设计**：如果因推理延迟不能增加层数，AttnRes 也能在相同深度下提升效果。
- **适合作为研究方向**的领域：残差连接的理论分析、深度-序列对偶性的进一步探索、以及更高效的深度注意力机制设计。
