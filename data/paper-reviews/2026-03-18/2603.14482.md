# V-JEPA 2.1: 解锁视频自监督学习中的稠密特征

> **论文信息**: arXiv:2603.14482 (2026年3月15日)
> **作者**: Lorenzo Mur-Labadia, Matthew Muckley, Amir Bar, Mido Assran 等 (Meta FAIR + 萨拉戈萨大学)
> **共同通讯作者**: Yann LeCun, Nicolas Ballas, Adrien Bardes
> **代码**: https://github.com/facebookresearch/vjepa2

---

## 一、背景与动机

视频自监督学习（Video SSL）近年来被视为构建世界模型的核心路径——模型需要从无标注的视觉数据中学习表征，使其能够感知、预测和规划物理世界中的行为。然而，一个关键难题始终未被解决：**如何同时获得高质量的稠密局部特征和强大的全局语义理解？**

具体而言，以 DINO/DINOv3 为代表的图像自监督方法擅长生成精细的稠密特征（适用于分割、深度估计等像素级任务），但缺乏对时序动态的建模能力。以 V-JEPA 2 为代表的视频自监督方法（基于联合嵌入预测架构 JEPA）在全局视频理解和动作预测上表现优异，但其学到的特征图却呈现出**碎片化的空间结构**——PCA 可视化显示特征图充满噪声，缺乏有意义的局部空间布局。用 V-JEPA 2 的特征做线性探测时，ADE20K 语义分割仅 22.2 mIoU，NYUv2 深度估计 RMSE 高达 0.682，与 DINOv3 的 55.9 mIoU 和 0.309 RMSE 形成鲜明对比。

本文的核心洞察来自一个简单但深刻的观察：V-JEPA 2 的预测损失**仅施加在被遮蔽（masked）的 token 上**，而可见的上下文 token 完全不受监督。这使得模型没有动力在上下文 token 中编码局部空间信息——它们更倾向于充当"全局聚合器"，把所有计算资源投入到聚合全局信息以最小化遮蔽区域预测误差上。

基于这一洞察，Meta FAIR 团队提出了 **V-JEPA 2.1**，通过一系列精巧的设计将预测损失扩展到所有 token，显著提升了稠密特征质量，同时保持了强大的全局理解能力。

## 二、核心方法

V-JEPA 2.1 的训练框架包含四个关键创新：

### 2.1 稠密预测损失（Dense Predictive Loss）

这是本文最核心的贡献。作者提出将自监督损失从仅覆盖遮蔽 token 扩展到覆盖**所有 token**（包括可见的上下文 token），总损失为：

$$\mathcal{L}_{\text{dense}} = \mathcal{L}_{\text{predict}} + \mathcal{L}_{\text{ctx}}$$

其中 $\mathcal{L}_{\text{ctx}}$ 对上下文 token 施加与遮蔽 token 相同的 L1 损失。但直接施加等权重的上下文损失会导致全局语义任务性能下降（模型可能找到"直接复制上下文特征"的捷径），因此作者设计了**距离加权方案**：

$$\lambda_i = \frac{\lambda}{\sqrt{d_{\text{min}}(i, M)}}$$

其中 $d_{\text{min}}(i, M)$ 是上下文 token $i$ 到最近遮蔽 token 的时空距离（以 patch 块数计）。这个设计很直观：离遮蔽区域越近的 token，其局部连续性约束越强；远离遮蔽区域的 token 权重自然衰减。此外，上下文损失系数还引入了渐进式 warm-up（从第 50 到第 100 epoch 逐渐增大），有效稳定了训练过程。

引入上下文损失后，ADE20K 分割从 22.2 跃升至 33.9 mIoU（+52%），NYUv2 深度 RMSE 从 0.682 降至 0.473（-31%），PCA 可视化也显示特征图从噪声碎片变为语义连贯的空间结构。

### 2.2 深度自监督（Deep Self-Supervision）

稠密预测损失的代价是全局任务性能下降（SSv2 从 72.8% 降至 62.5%）。为解决这一矛盾，作者提出了深度自监督：将预测损失层级式地施加到编码器的**多个中间层**上。具体做法是拼接 4 个中间层的输出，经 MLP 融合后输入预测器，预测器同时在 4 个层级产生输出并计算损失。

这一设计让局部信息能够从浅层流向深层，同时恢复了全局语义理解能力。最终 SSv2 恢复到 72.0%，ImageNet 达到 80.8%，而稠密任务进一步提升至 38.6 mIoU（ADE20K）和 0.463 RMSE（NYUv2）。消融实验还表明，深度自监督使得下游评估不再需要多层特征拼接，仅使用最后一层即可获得优异性能。

### 2.3 多模态分词器（Multi-Modal Tokenizer）

V-JEPA 2 用统一的 3D 卷积处理图像和视频，将图像复制为 16 帧静态视频——这既增加了计算开销，又引入了不合理的表征偏差。V-JEPA 2.1 为图像和视频分别设计了 2D 卷积（16×16）和 3D 卷积（16×16×2）的分词器，并添加可学习的模态嵌入 token，使模型能够区分静态外观和时序运动信息。

### 2.4 数据与模型扩展

- **数据扩展**：构建 VisionMix-163M 数据集，将图像从 1M（ImageNet）扩展到 142M（LVD-142M），同时调整视频采样权重偏向动态内容丰富的 YT-1B
- **模型扩展**：从 ViT-g（300M）扩展到 ViT-G（2B）
- **高分辨率冷启动（Cooldown）**：第二阶段将视频从 16 帧/256² 提升至 64 帧/384²，图像从 256² 提升至 512²，同时衰减学习率

## 三、实验设计

论文的实验设计非常全面，覆盖了五大类下游任务：

**预测性任务**：
- Ego4D 短期物体交互预测（STA）：预测交互位置、物体类别、动作和时间
- EPIC-KITCHENS-100 动作预测：预测未来 1 秒的动词、名词和联合动作

**世界模型任务**：
- 真实机械臂抓取（Franka Panda，零样本部署）
- 潜在空间导航规划（Tartan Drive 等）

**稠密视觉任务**：
- 单目深度估计（NYUv2, KITTI，线性探测）
- 语义分割（ADE20K, Cityscapes, VOC12，线性探测）
- 视频目标分割（DAVIS, YouTube-VOS，无参数标签传播）

**全局理解任务**：
- 动作识别（SSv2, Diving-48, K400，注意力探测）
- 图像分类（ImageNet）
- 视频问答（PerceptionTest, MVP, MVBench 等）

**蒸馏实验**：将 ViT-G（2B）蒸馏为 ViT-L（300M）和 ViT-B（80M）

所有评估均采用冻结编码器 + 轻量探测头的协议，保证了公平对比。

## 四、结果与分析

### 4.1 预测性任务——全面超越

在 Ego4D STA 上，V-JEPA 2.1 ViT-G 达到 **7.71 mAP**，比之前最优方法 STAformer（5.67 mAP）提升约 35%。这一优势主要来自对时间和空间维度的精确预测（$\delta$ 时间预测 AP 20.2，检测 AP 50.7）。在 EPIC-KITCHENS 动作预测上达到 **40.8 Recall@5**，相比 V-JEPA 2 的 39.7 提升 +2.8%，创造了该任务的绝对 SOTA。

### 4.2 世界模型任务——从特征到行动

V-JEPA 2.1 的稠密特征直接转化为真实机器人的操控能力：在 Franka Panda 机械臂上零样本部署，抓取成功率从 V-JEPA 2 的 60% 提升至 **80%**（+20%），且更长的规划时序不会导致性能退化（V-JEPA 2 反而退化）。在导航任务中，V-JEPA 2.1 在 Tartan Drive 上实现 5.687 ATE（优于 NWM 的 5.831），同时规划速度提升 **10 倍**（10.6s vs 103.2s），仅需 8 步去噪而非 SD-VAE 的 128 步。

### 4.3 稠密视觉任务——从"不可用"到 SOTA

深度估计是最大的亮点：V-JEPA 2.1 ViT-G 在 NYUv2 上达到 **0.307 RMSE**，超越了 70 亿参数的 DINOv3（0.309），是所有小于 2B 参数模型中的绝对最优。相比 V-JEPA 2 的 0.642，提升幅度高达 52%。语义分割方面，V-JEPA 2.1 在 VOC12 上达到 85.0 mIoU，在 Cityscapes 上达到 73.5 mIoU，虽然略逊于 DINOv3（86.6, 81.1），但考虑到 V-JEPA 2.1 同时具有强大的时序建模能力，这一权衡是合理的。

### 4.4 全局任务——保持领先

SSv2 动作识别达到 **77.7%** Top-1，超越 InternVideo2 全量微调的 77.5% 和 DINOv3 7B 的 70.1%。ImageNet 达到 85.5%，K400 达到 87.7%，均在 ViT 级别模型中保持竞争力。

### 4.5 蒸馏效果显著

从 ViT-G 蒸馏到 ViT-L 后，SSv2 从 74.2% 提升至 76.5%（接近 ViT-G 的 77.7%），ADE20K 从 42.0 提升至 46.7 mIoU，KITTI 深度 RMSE 达到 2.490（接近 ViT-G 的 2.461），证明了大模型到小模型的知识迁移非常高效。

## 五、评价与思考

### 亮点

1. **核心洞察精准且优雅**：仅将损失从"仅遮蔽 token"扩展到"所有 token"就带来了质的飞跃，说明自监督学习中"哪些 token 受监督"这一设计选择被长期忽视
2. **全局与局部的统一**：通过深度自监督巧妙解决了稠密特征与全局理解之间的 trade-off，而非简单取舍
3. **全方位验证**：从特征可视化到线性探测，从预测任务到真实机器人部署，证据链完整且令人信服
4. **务实的数据策略**：复用 LVD-142M + YT-1B 构建 VisionMix-163M，避免了大规模数据收集的成本

### 值得关注的问题

1. **语义分割仍有差距**：在 ADE20K（47.9 vs DINOv3 的 55.9）和 Cityscapes（73.5 vs 81.1）上差距明显。作者推测 VisionMix 中缺乏足够复杂的场景，但更深层次的原因可能是 JEPA 框架的遮蔽预测目标天然偏向局部连续性，而非类别级语义区分
2. **VideoQA 表现波动**：在 TemporalBench 和 TOMATO 上反而弱于 V-JEPA 2，说明稠密特征增强并未均匀惠及所有下游任务
3. **计算成本**：2B 模型 + 64 帧 + 384² 分辨率 + 24 层预测器的训练开销相当可观，论文未详细报告
4. **与 DINOv3 的互补性**：两者各有所长（V-JEPA 2.1 擅长时序+稠密，DINOv3 擅长极致稠密+7B 规模），是否可以通过蒸馏融合？

### 更深远的意义

V-JEPA 2.1 的成功为"世界模型"路线提供了有力支持：更好的稠密表征直接转化为更好的机器人操控和导航能力。这暗示着，构建通用具身智能体的关键基础可能不是更大的语言模型，而是更精细的视觉-时序表征。

## 六、适用场景

1. **具身智能与机器人**：需要精确深度估计和时空理解的抓取、导航、操作任务，V-JEPA 2.1 的零样本迁移能力尤其有价值
2. **视频预测与规划**：Ego4D STA 和 EPIC-KITCHENS 的 SOTA 表明，适用于需要预测人类行为和物体交互的应用（如自动驾驶中的行人意图预测、AR/VR 中的交互预测）
3. **稠密视觉理解**：深度估计、语义分割、视频目标分割等需要像素级特征的任务，特别是同时需要时序一致性的场景
4. **视频大模型骨干网络**：作为视频 LLM 的视觉编码器，V-JEPA 2.1 的特征已在 VideoQA 中展示了竞争力
5. **资源受限部署**：通过蒸馏得到的 ViT-B（80M）和 ViT-L（300M）版本在保持竞争力的同时大幅降低推理成本
