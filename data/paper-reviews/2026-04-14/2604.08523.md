# 论文分析：ClawBench —— 跨越沙盒，AI 智能体挑战真实世界的“终极考场”

## 1. 标题
*   **原标题：** ClawBench: Can AI Agents Complete Everyday Online Tasks?
*   **中文翻译：** ClawBench：AI 智能体能完成日常在线任务吗？

## 2. 论文信息
*   **作者：** Yuxuan Zhang, Yubo Wang, Yipeng Zhu, Penghui Du, Junwen Miao, Xuan Lu, Wendong Xu, Yunzhuo Hao, Songcheng Cai, Xiaochen Wang, Huaisong Zhang, Xian Wu, Yi Lu, Minyi Lei, Kai Zou, Huifeng Yin, Ping Nie, Liang Chen, Dongfu Jiang, Wenhu Chen, Kelsey R. Allen
*   **主要机构：** 不列颠哥伦比亚大学 (UBC)、Vector Institute、滑铁卢大学、多伦多大学、卡内基梅隆大学、上海交通大学、浙江大学、香港科技大学、清华大学、Netmind.ai、UniPat AI、Etude AI
*   **arXiv 链接：** [https://arxiv.org/abs/2604.08523](https://arxiv.org/abs/2604.08523)
*   **项目主页：** [https://claw-bench.com](https://claw-bench.com)

## 3. 背景与动机
随着大语言模型（LLM）能力的飞速提升，AI 智能体（AI Agents）开始展现出自主导航图形界面、填写表单和执行多步工作流的潜力。然而，一个核心问题始终悬而未决：**这些智能体在多大程度上能真正成为日常生活的通用助手？**

现有的 Web 智能体基准测试（如 WebArena, OSWorld, TheAgentCompany）大多存在以下局限性：
1.  **沙盒化局限：** 它们通常运行在离线沙盒或静态 HTML 页面中，去除了 Cookie 弹窗、动态 JavaScript 渲染、身份验证流和反爬虫机制等真实世界的复杂性。
2.  **“只读”偏向：** 现有的基准测试多侧重于信息检索（QA），而对于真正能“改变状态”的写入密集型任务（如订票、提交申请、购物）涉及较少。
3.  **安全性与真实性的权衡：** 在真实生产网站上测试写入任务存在安全风险（如误下单扣费），这导致研究者不得不退守沙盒，从而失去了生态有效性。

**ClawBench 的诞生正是为了打破这一僵局**。它旨在提供一个包含真实世界复杂度、支持写入操作且具备安全保障的评估框架。

## 4. 核心方法
ClawBench 的核心在于其**“真实环境 + 安全拦截”**的创新架构：

### 4.1 任务设计与拦截机制
*   **153 个真实任务：** 涵盖 144 个在线平台，分为 8 个大类（日常、工作、开发、社交、学术、旅行、宠物、金融）。
*   **HTTP 级拦截技术：** 这是一个关键创新。ClawBench 使用轻量级 Chrome 扩展程序和 CDP 协议，实时监控网络流量。它只拦截**最终的提交请求**（即导致不可逆交易的那个 HTTP 调用），而允许页面加载、AJAX 渲染等良性请求正常通过。这确保了智能体在真实生产网站上运行，但不会产生真实的副作用（如不会真的下一张订单）。

### 4.2 五层数据记录基础设施
为了实现深度的失败诊断，ClawBench 记录了智能体运行时的五层同步数据：
1.  **会话录像：** 完整的浏览器操作视频。
2.  **动作截图：** 每一交互步骤后的截图。
3.  **HTTP 流量：** 包含请求体、Payload 和 Header 的详细记录。
4.  **智能体消息：** 记录推理链（Chain-of-Thought）和工具调用。
5.  **浏览器底层动作：** 精确的鼠标坐标、按键流和滚动偏移。

### 4.3 智能体化评估器（Agentic Evaluator）
ClawBench 引入了基于 Claude Code 的“智能体评估器”。它不再仅仅检查最终状态，而是将智能体的执行轨迹与人类标注者的**真理轨迹（Ground-truth Trajectory）**进行多维度对比。评估器会检查每一个必填字段、交互顺序和终端 Payload 结构，从而给出带有结构化理由的二进制（通过/失败）判定。

## 5. 实验设计
研究团队评估了 7 款当前最顶尖的 AI 模型，涵盖闭源和开源阵营：
*   **闭源模型：** Claude Sonnet 4.6, GPT-5.4, Gemini 3.1 Flash Lite, Claude Haiku 4.5, Gemini 3 Flash。
*   **开源模型：** GLM-5, Kimi K2.5。

**实验环境：** 每个模型通过 OpenClaw 框架控制 Chromium 浏览器，处于完全隔离的容器中。所有模型均在相同的五个数据层记录下运行，以确保可复现性和公平性。

## 6. 结果与分析
实验结果令人警醒，揭示了当前 AI 智能体在面对真实世界挑战时的巨大鸿沟：

1.  **成功率普遍偏低：** 即使是表现最好的 **Claude Sonnet 4.6，成功率也仅为 33.3%**。
2.  **“沙盒泡沫”破裂：** 在传统的 WebArena 或 OSWorld 基准上，Claude 3.5/GPT-4 级别的模型通常能达到 65-75% 的高分。但在 ClawBench 中，GPT-5.4 的得分竟然跌至 **6.5%**。这表明：**在受控环境下表现优秀的智能体，一旦进入充满变数的真实 Web 环境，其鲁棒性会剧烈下降。**
3.  **模型差异化显著：**
    *   **Claude Sonnet 4.6** 在日常、金融和学术场景中领先。
    *   **GLM-5**（清华系）表现亮眼，在“工作”类别任务中排名第一，总分 24.2% 位居第二。
    *   **Gemini 3 Flash** 在旅行规划场景中表现较好。
4.  **失败模式分析：** 失败主要源于对动态 DOM 结构解析错误、无法正确处理复杂的文档提取、以及在多步骤长路径任务中的指令漂移。

## 7. 评价与思考（深度见解）

### 7.1 从“玩具环境”到“真实战场”的跃迁
ClawBench 的重要意义在于它重新定义了 Web 智能体的评测标准。过去我们满足于智能体在 5 个自建网站（WebArena）上刷分，而 ClawBench 将其丢到了 144 个时刻变化的真实网站中。这种**真实感**是通往通用人工智能（AGI）的必经之路。

### 7.2 安全拦截：一种巧妙的中间路线
论文提出的 HTTP 请求级拦截机制非常巧妙。它解决了“炸实验室”的难题：既不需要维护昂贵的镜像网站，也不需要面临真实的法律和经济风险。这种方法可以被其他领域（如本地文件系统、数据库操作）的智能体评估所借鉴。

### 7.3 为什么 GPT-5.4/Claude 4.6 依然会失败？
深层次的原因可能在于：
*   **跨文档推理能力：** 日常任务通常需要读取用户提供的 PDF、邮件或 Excel 片段，并将其准确填入网页。当前模型在多模态对齐（截图 vs 网页代码 vs 用户文档）上仍有偏差。
*   **动态交互的脆弱性：** 真实网页有大量的异步加载和反爬校验。智能体习惯于静态观察，难以应对“点击后需等待 2 秒且 DOM 结构改变”的动态环境。
*   **缺乏容错纠错机制：** 当智能体填错一个字段被网页报错时，它们往往陷入无限循环或直接崩溃，缺乏像人类一样的自愈能力。

### 7.4 可追溯性的价值
论文强调的“五层记录”不仅是为评估服务，更是为开发者提供了珍贵的 Debug 信号。未来的智能体开发将不再是盲目地增加模型参数，而是需要针对这些详细的失败轨迹进行“场景化强化训练”。

## 8. 适用场景
ClawBench 框架及其数据对于以下领域具有极高的应用价值：
1.  **个人助理开发：** 用于测试自动订票、报销、申请等功能的可靠性。
2.  **流程自动化（RPA）升级：** 下一代基于 LLM 的 RPA 需要在不写死规则的情况下适应网页更新，ClawBench 是最佳的测试床。
3.  **模型能力诊断：** 评估模型在复杂逻辑推理、视觉空间理解和多步决策上的综合表现。
4.  **智能体安全研究：** 研究拦截机制如何保护生产系统，防止 AI 误操作导致的数据污染或经济损失。
