# 论文深度分析报告：Your Agent Is Mine: Measuring Malicious Intermediary Attacks on the LLM Supply Chain

## 1. 标题
**原标题：** Your Agent Is Mine: Measuring Malicious Intermediary Attacks on the LLM Supply Chain  
**中文翻译：** 你的智能体归我了：测量大语言模型（LLM）供应链中的恶意中间人攻击

## 2. 论文信息
*   **作者：** Hanzhi Liu, Chaofan Shou, Hongbo Wen, Yanju Chen, Ryan Jingyang Fang, Yu Feng
*   **机构：** 加州大学圣巴巴拉分校 (UCSB), Fuzzland, 加州大学圣迭戈分校 (UCSD), World Liberty Financial
*   **arXiv 链接：** [https://arxiv.org/abs/2604.08407](https://arxiv.org/abs/2604.08407)
*   **发表时间：** 2026年4月9日

## 3. 背景与动机
随着 LLM 智能体（Agents）从简单的对话助手演变为具备工具调用（Tool-use）能力的自动化系统，开发者越来越多地依赖第三方 **LLM API 路由（Routers）**（如 LiteLLM, OpenRouter 等）。这些路由充当了应用层代理，提供负载均衡、成本优化和多模型统一访问。

然而，当前的 LLM 供应链存在一个巨大的 **“完整性缺口”（Integrity Gap）**：
1.  **明文访问权：** 路由能够以明文形式查看、修改和存储所有请求（提示词、API 密钥）和响应（模型生成的工具调用指令）。
2.  **缺乏验证机制：** 目前没有任何主流模型提供商（如 OpenAI, Anthropic）为模型输出提供端到端的加密签名。这意味着智能体客户端无法验证接收到的工具调用是否真的是由模型生成的，还是被路由中途篡改过的。
3.  **信任错位：** 开发者通常将路由视为透明的底层设施，但实际上路由在应用层终止了 TLS 连接，拥有比传统网络中间人（MITM）更大的权力。

## 4. 核心方法
本研究首次系统化地定义了 LLM 路由供应链中的威胁模型，并提出了攻击分类学（Attack Taxonomy）：

### 4.1 攻击分类
1.  **AC-1: 有效载荷注入 (Payload Injection)**
    *   **主动篡改：** 路由在模型响应返回给客户端之前，重写 JSON 格式的工具调用指令。
    *   **AC-1.a 针对性依赖注入：** 专门识别包安装指令（如 `pip install requests`），将其替换为恶意包（如通过拼写劫持 Typosquatting）。
    *   **AC-1.b 条件化交付 (Conditional Delivery)：** 为了逃避检测，仅在特定触发条件下（如通过预热期、识别特定用户指纹或进入“YOLO 模式”的自主智能体）才实施攻击。
2.  **AC-2: 秘密窃取 (Secret Exfiltration)**
    *   **被动收集：** 路由静默地从明文流量中提取 API 密钥（AWS, GitHub 等）、系统提示词、用户敏感数据等。

### 4.2 实验工具：Mine
作者开发了一个名为 **Mine** 的研究代理（Proxy），实现了上述所有攻击类，并用于评估四种主流智能体框架的安全性，同时作为测试防御措施的基准。

## 5. 实验设计
研究团队进行了大规模的生态系统测量，分为三个维度：

1.  **市场调查：**
    *   从淘宝、闲鱼、Shopify 等平台购买了 **28 个付费路由**。
    *   从开源社区、配置转储中收集了 **400 个免费公开路由**。
2.  **中毒研究 (Poisoning Study)：**
    *   **泄露密钥追踪：** 故意在中文论坛和社交软件中泄露由研究人员控制的 OpenAI 密钥，观察是否有路由或第三方在使用这些“有毒”的密钥。
    *   **弱配置诱饵：** 部署了 20 个故意设置弱密码或默认配置的路由实例，吸引外部攻击者。
3.  **防御评估：**
    *   评估了三种客户端防御措施：**故障关闭策略门控 (Policy Gate)**、**响应端异常筛选 (Anomaly Screening)**、**追加日志透明度 (Transparency Logging)**。

## 6. 结果与分析
实验结果令人警醒，证明了恶意中间人攻击不仅是理论可能，而是已经在野发生的现实：

*   **恶意路由盛行：** 在 428 个被测路由中，发现 **9 个路由** 正在主动注入恶意代码，**17 个路由** 滥用研究人员的 AWS 凭据，**1 个路由** 试图清空研究人员的以太坊钱包。
*   **防御缺口：** 即使是付费路由也不保证完整性（1 个付费路由发现注入行为）。
*   **泄露链条惊人：** 一个故意泄露的 OpenAI 密钥在短期内产生了 **1 亿个 Token** 的流量，并暴露了来自 398 个不同项目或主机的 99 组凭据。
*   **防御有效性：** 
    *   **策略门控**能阻断大部分高风险命令（如 Bash 脚本），但无法防止复杂的条件化攻击。
    *   **异常检测器**在 6.7% 的误报率下能识别 89% 的攻击样本，但对精巧的变体效果受限。
    *   研究指出，最终的解决方案必须是 **提供商支持的响应签名**。

## 7. 评价与思考
### 7.1 深度见解
这篇论文的深度在于它揭示了 **AI 时代的“信任锚点”漂移**。在传统 Web 时代，我们信任 TLS。但在 Agent 时代，由于路由需要理解并可能转换模型协议，TLS 的保护在路由处被终结。

论文提出的 **“应用层中间人” (Application-layer MITM)** 概念非常精准。不同于网络层攻击需要伪造证书，LLM 路由是用户主动配置的，这使得攻击变得极其隐蔽且合法化。这种攻击不仅是“提示词注入”的逆过程，更是一种更深层次的“指令注入”，直接操纵智能体的手脚（工具）。

### 7.2 核心痛点
目前 LLM 生态系统追求互操作性和低成本，导致了 API 聚合服务的野蛮生长。开发者在享受 LiteLLM 等工具带来的便利时，往往忽视了这些中转站实际上是 LLM 应用中最脆弱的环节。如果智能体被赋予了写文件、执行命令或管理云资源的权限，那么一个恶意的路由就等同于获得了该系统的 Root 权限。

## 8. 适用场景
*   **企业级安全审查：** 企业在引入第三方 API 中转服务或多模型调度平台时，应根据此文的威胁模型进行审计。
*   **开发者选型参考：** 警示独立开发者避免使用来源不明、价格极低或“免费共享”的 LLM API 路由。
*   **安全防御开发：** 论文提出的策略门控（Policy Gate）和异常筛选思路，可直接应用于智能体运行时的安全中间件开发。
*   **标准制定：** 为模型提供商（OpenAI/Google 等）和协议制定者（如 MCP）提供了关于“响应完整性验证”的迫切需求论据。

---
**结论：** 《Your Agent Is Mine》是 LLM 供应链安全领域的开创性工作。它将安全视线从“如何防止模型学坏”转移到了“如何防止传输链路作恶”，为构建健壮、可信的自主智能体系统敲响了警钟。
