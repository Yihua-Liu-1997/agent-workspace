# 📄 PRISM: 解密中期训练中的知识保留与交互

**原文标题**: PRISM: Demystifying Retention and Interaction in Mid-Training

**作者**: Bharat Runwal, Ashish Agrawal, Anurag Roy, Rameswar Panda

**机构**: IBM Research (从模型选择推断)

**arXiv**: [2603.17074](https://arxiv.org/abs/2603.17074)

**点赞**: 93 | **发布日期**: 2026-03-17

## 背景与动机

大语言模型（LLM）的训练通常分为三个阶段：预训练（Pre-training）、中期训练（Mid-training）和后训练（Post-training，包括 SFT 和 RL）。其中，预训练赋予模型广泛的语言知识，后训练通过指令微调和强化学习（RL）使其具备特定能力。然而，**中期训练**——介于预训练和后训练之间的关键阶段——长期以来缺乏系统性的实证研究。

当前社区面临的核心困惑是：中期训练到底在模型能力提升中扮演什么角色？它与后续的 RL 阶段如何相互作用？为什么有些模型在直接进行 RL 后能大幅提升推理能力，而另一些模型几乎毫无反应？这些问题的答案不仅关乎理论理解，更直接影响工业界如何设计高效、可靠的训练管线。

PRISM 正是针对这些空白而诞生的。它不仅是一个方法框架，更是一项**大规模系统性实证研究**，旨在回答中期训练中的核心设计问题：数据怎么配、时机怎么选、与 RL 如何协同。

## 核心方法

PRISM 的核心思路可以概括为一句话：**通过约 27B 高质量 token 的中期训练，为后续 RL 阶段搭建"能力地基"**。

### 实验设计矩阵

PRISM 的实验规模令人印象深刻，涵盖了极其丰富的变量空间：

- **7 个基座模型**，横跨 **4 个模型家族**：Granite、LLaMA、Mistral、Nemotron-H
- **2 种架构类型**：稠密 Transformer 和 Attention-Mamba 混合架构
- **参数规模**从 3B 到 24B
- **3 种数据配置**：Math-only、Math+Code、Math+Code+Science，逐步丰富

### 数据策略

数据是中期训练的核心。PRISM 使用了精心筛选的高质量数据集：

- **通用数据**：DCLM-EDU 语料（质量分 ≥ 3），提供广泛的通用知识基础
- **数学推理**：OpenCodeReasoning-2（经 QwQ 评估器过滤，保留正确样本，Python 和 C++ 各 60K 样本）
- **代码**：包含多种编程语言和竞赛题的数据集
- **科学推理**：高质量的科学问答数据
- **对话数据**：WildChat-1M（限制为 GPT-4 生成的高质量对话），经过标准化格式处理

数据配置采用**渐进式混合策略**：从 Math-only 开始，逐步加入 Code 和 Science 数据。通过在不同阶段实验不同比例的数据混合，PRISM 系统性地揭示了数据组成对最终性能的决定性影响。

### 关键超参发现

- **上下文长度**：中期训练在 8K 上下文长度时效果最佳，继续增加收益递减
- **训练稳定性**：适当的数据配比可以有效防止对特定领域的过拟合

## 实验设计

### 评测基准

PRISM 在多个维度上全面评测模型能力：

- **数学**：AIME'24、AIME'25、MATH500
- **代码**：LiveCodeBench、Codeforces
- **科学推理**：GPQA-Diamond
- **通用推理**：综合平均分

### 核心对比实验

实验设计的精妙之处在于设置了多组消融对比：

1. **Base → RL vs. Base → Mid-train → RL**：直接 RL 与经过中期训练后再 RL 的对比
2. **不同数据组成的中期训练**：Math-only vs. Math+Code vs. Math+Code+Science
3. **不同 RL 数据混合**：在相同中期训练基础上，调整 RL 阶段的数据配比
4. **不同模型架构**：验证发现的跨架构普适性

## 结果与分析

### 1. 中期训练带来显著且一致的能力提升

在约 27B 高质量 token 的中期训练后：

- **数学基准**：+15 到 +40 分的提升
- **代码基准**：+5 到 +12 分的提升
- **科学基准**：+6 到 +13 分的提升

最关键的是，这些提升在**保持通用能力不退化**的前提下实现，体现了"保留"（Retention）的核心价值。

### 2. PRISM → RL 管线效果惊人

这是论文最震撼的发现：

- **完整 PRISM → RL 管线**：将六个推理基准的宏观平均分从不到 12 分提升到 **29-42 分**（3-4 倍提升）
- **直接在基座模型上做 RL**：大多数基座模型的 AIME 分数**接近零**，几乎没有推理能力

这表明中期训练不是锦上添花，而是 RL 能够生效的**必要前提条件**。

### 3. 数据组成在中期训练阶段最为关键

论文揭示了一个反直觉的发现：

- 在中期训练中包含科学数据，可以在 RL 阶段解锁 **+17 到 +28 分**的 GPQA-Diamond 增益
- 而在 RL 阶段改变数据混合比例，仅产生**不到 2 分**的差异

这说明：**中期训练决定了模型的能力天花板，RL 只是在这个天花板内进行优化**。

### 4. 机制层面的深度洞察

PRISM 通过权重分析和 CKA（Centered Kernel Alignment）表征分析，揭示了中期训练与 RL 在机制层面的本质区别：

- **中期训练**：密集地重构了超过 **90%** 的模型权重——这是一种深层的、全局性的知识重组
- **RL**：仅对约 **5%** 的参数进行稀疏的、集中在前层的精细调整
- **表征保持**：RL 一致地保持了中期训练建立的表征几何结构（CKA > 0.998），跨架构均成立

最引人深思的发现是：**无论从哪个起点开始，RL 施加的权重变化几乎是相同的，但它只在经过中期训练的模型上才有效**。这有力地支持了一个观点：中期训练将模型推入了一个特定的"能力配置空间"，使得后续的 RL 能够在这个空间中有效优化。

## 评价与思考

### 这篇论文的价值

PRISM 的核心贡献不在于提出了某个新算法，而在于通过**大规模、系统性的实证研究**澄清了一个困扰社区已久的关键问题：中期训练为什么重要，以及它如何与 RL 协同工作。

几个特别值得关注的 Insight：

1. **"地基假说"得到实证支持**：中期训练不是简单的"多训一些数据"，而是为 RL 搭建了必要的表征基础。没有这个地基，RL 的梯度信号无法被模型有效利用。这解释了为什么社区中有些 RL 实验效果惨淡——问题可能不在 RL 本身，而在基座模型的准备不足。

2. **数据组成的非对称重要性**：中期训练的数据组成远比 RL 阶段重要。这对工业实践有直接的指导意义——与其花大量精力调优 RL 阶段的数据配比，不如把资源投入到中期训练的数据筛选和配比上。

3. **权重层面的互补机制**：90% vs 5% 的权重变化对比，揭示了一种优雅的分工——中期训练负责"建立能力框架"，RL 负责"精细调优"。这种互补性使得整个管线既高效又稳定。

### 深层思考

论文中最深刻的发现——"RL 施加相同的权重变化但只在中期训练后的模型上有效"——实际上暗示了一个关于**损失景观（Loss Landscape）**的重要洞见：中期训练可能将模型从预训练损失景观的一个区域转移到了另一个更有利于 RL 优化的区域。在这个新区域中，相同的 RL 梯度信号能被有效利用；而在原始区域中，这些信号可能被噪声淹没或导致次优解。

这也引出了一个有趣的理论问题：是否存在一种"最优的中期训练终态"，使得后续 RL 的效果最大化？如果存在，我们能否通过某种代理指标（如 CKA、特定层的激活模式等）来在线检测这个终态？

### 局限性

- 论文主要聚焦于 3B-24B 的模型规模，对于 70B+ 大模型的结论是否一致尚待验证
- 中期训练的具体数据配比策略仍需要更多消融实验来细化
- 对"为什么中期训练后的表征更适合 RL"这一问题，论文给出了现象描述但尚未提供充分的理论解释

## 适用场景

PRISM 的发现对以下场景具有直接的指导价值：

1. **LLM 训练管线设计**：任何计划将 RL 作为后训练核心环节的团队，都应重视中期训练的设计。论文提供了清晰的数据配比建议和实验验证框架。

2. **推理能力增强**：如果你的目标是提升模型在数学、代码、科学推理等任务上的表现，PRISM 的中期训练策略提供了一个经过验证的路线图。

3. **模型架构研究**：论文跨架构的实验结果（包括 Attention-Mamba 混合架构）为不同架构的训练策略提供了参考。

4. **数据工程优先级**：对于资源有限的团队，PRISM 明确指出应优先投入中期训练的数据质量，而非过度调优 RL 阶段。

---

**总结**：PRISM 是一篇高质量的实证研究论文，通过精心设计的实验矩阵，系统性地揭示了中期训练与 RL 之间的协同机制。它不仅提供了实用的训练指南，更重要的是加深了我们对 LLM 训练过程中知识保留与交互的理解。对于任何从事 LLM 训练的研究者和工程师来说，这都是一篇值得仔细研读的论文。
