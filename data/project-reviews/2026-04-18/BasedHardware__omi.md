# BasedHardware/omi：你的开源 AI "第二大脑"与全天候记忆助手

## 项目简介
BasedHardware 的 **omi**（前身为 Friend）是一款开源的 AI 可穿戴设备与软件生态系统。它的核心理念是成为一个“比你自己的大脑更值得信赖的第二大脑”。通过持续捕获你的屏幕画面和日常对话，它能够在本地或云端进行实时语音转录，自动生成对话摘要、提取待办事项（Action Items），并提供一个“记住你所看、所听一切”的 AI 聊天交互接口。

该项目致力于解决现代人在高信息密度环境下容易遗忘重要细节、缺乏高效知识沉淀工具的问题。区别于市面上动辄收取高昂订阅费的闭源硬件（如 Humane Ai Pin 或 Limitless），omi 提供了**从硬件 PCB 图纸到固件，再到移动端、桌面端及后端代码的完全开源方案**。

## 核心特性与技术架构

### 核心特性
1. **全天候记录与转录**：支持 24 小时以上的音频持续捕获与实时转录，内置说话者分离（Speaker Diarization）技术，能够识别并区分不同的说话人。
2. **多端支持与跨平台**：提供适配 iOS / Android 的移动端应用，以及 macOS 的桌面端应用，覆盖用户的生活与工作场景。
3. **完全开源与隐私可控**：用户可以选择使用官方的开箱即用云服务，也可以完全本地化部署后端服务，将数据牢牢掌握在自己手中，彻底消除隐私顾虑。
4. **插件与应用生态（MCP）**：支持模型上下文协议（MCP），允许通过应用市场集成 GitHub、Slack、Notion、Google Calendar 等第三方效率工具，实现自动化工作流。

### 技术架构
Omi 的技术栈非常现代且多样化，是一套完整的软硬件结合系统：
- **硬件层（Firmware/Device）**：采用 Nordic nRF5340（低功耗双核蓝牙 SoC）或 ESP32-S3，配备高灵敏度双麦克风阵列，通过 BLE（低功耗蓝牙）与手机进行持续的数据流传输。
- **移动端（Mobile App）**：基于 **Flutter** 构建跨平台应用（iOS & Android），负责接收可穿戴设备传来的蓝牙音频流，并与云端进行 HTTPS/WebSocket 通信。
- **桌面端（macOS App）**：采用 **Swift/SwiftUI** 构建原生 UI，底层结合 **Rust** 开发高性能本地后端，处理屏幕画面捕获与桌面音频处理。
- **后端服务（Backend API）**：基于 **Python** 与 **FastAPI** 搭建，集成 Pusher (WebSocket) 处理实时推送。
- **AI 与数据管道**：
  - **语音识别**：使用 VAD (语音活动检测) 过滤静音，集成 Deepgram 等外部 API 进行高精度语音转文本 (STT)。
  - **存储与缓存**：使用 Firebase / Firestore 作为主数据库，Redis 作为缓存引擎，Pinecone 作为向量数据库存储长期记忆，以供 RAG（检索增强生成）调用。
  - **大语言模型（LLM）**：支持灵活对接 GPT-4o、Claude 3.5 或本地运行的模型（如 Ollama），负责总结对话和提供 AI 助理问答。

## 适用场景与目标用户

### 适用场景
- **会议与头脑风暴**：自动记录长时间会议的完整内容，提取核心结论和待办事项，让参与者专注交流而非手动做笔记。
- **日常灵感与语音备忘**：在散步、通勤或与他人交谈时，无感记录一闪而过的灵感，并通过后续的 AI 对话找回细节。
- **个人知识库自动构建**：将日常碎片化的交谈、阅读的屏幕文档自动聚合并结构化，形成个人的动态维基（Personal Wiki）。

### 目标用户
- **极客与开发者**：喜欢折腾开源硬件、热衷于搭建私有化云服务、对数据隐私有极高要求的技术爱好者。
- **效率追求者**：高管、产品经理、研究员等需要处理海量信息、极度依赖高效笔记系统的人群。
- **记忆力辅助需求者**：希望利用 AI 作为外脑，辅助记忆日常繁杂细节的用户。

## 与同类项目对比

市场上已有数款知名的 AI 可穿戴设备，如 Humane Ai Pin、Rabbit R1、Limitless (前 Rewind) 和 Plaud Note。

- **相比 Humane Ai Pin / Rabbit R1**：Omi 并不试图“取代”智能手机，而是老老实实地作为手机的“外设传感器”存在。这使得其硬件成本极低（约 $89），且规避了独立运算带来的续航崩溃和严重发热问题。
- **相比 Limitless / Plaud Note**：Omi 的最大杀手锏是 **完全开源** 和 **零订阅费**。Limitless 和 Plaud 通常需要支付每月 $10-$20 的订阅费以解锁高级大模型总结功能。而 Omi 允许用户绑定自己的大模型 API Key，甚至连接本地私有化部署的免费模型，长远来看边际成本极低。
- **劣势与挑战**：作为开源驱动的硬件项目，Omi 在 App 交互的精致度、初次配对的稳定性上，可能暂时不如 Plaud 这类纯商业化产品成熟；此外，由于采用开放式环境麦克风，在极其嘈杂的公共场合，其降噪和收音效果依然依赖于算法的持续调优。

## 评价与思考

Omi 项目是开源社区对“AI 硬件化”浪潮的一次精彩反击。它用行动证明了：**当前阶段的 AI 可穿戴设备不需要是价格高昂、故弄玄虚的“下一代计算平台”，它最合理的形态就是一个纯粹的、低功耗的数据采集终端。**

从架构上看，Omi 采取了非常务实的“瘦终端、胖手机/云”策略。可穿戴设备只负责收集音频并无脑通过低功耗蓝牙推流，把极其耗电的网络通信、AI 推理和数据存储全部转交给用户的手机和云端。这种解耦设计不仅延长了续航，还让硬件迭代的压力大幅降低。

更难能可贵的是它极致的开源精神——开放 PCB 硬件图纸和全栈代码，极大地降低了开发者参与 AI 硬件创新的门槛。这种 Hacker-first 的理念为其在 GitHub 上赢得了接近 10k 的 Stars。

然而，所有“Always-On”的 AI 可穿戴设备都面临着同一个终极拷问：**社会接受度与隐私边界**。一个挂在胸前、持续录音的设备，在社交场合难免引起他人的防备甚至反感。Omi 未来的破局点不仅在于提高 STT 准确率，更在于如何通过精妙的工业设计（例如更隐蔽的外观、清晰的硬件级隐私断电开关）和社交契约的重塑，来化解这种科技带来的社交摩擦。

## 上手建议

如果你对 Omi 感兴趣，无论是否拥有硬件，都可以分阶段体验：

1. **纯软件体验（桌面端，零成本）**：
   对于 macOS 用户（需 macOS 14+ 及 Xcode），可以直接体验其屏幕捕获与记忆功能。打开终端运行：
   ```bash
   git clone https://github.com/BasedHardware/omi.git && cd omi/desktop && ./run.sh --yolo
   ```
   这会自动编译并在本地启动桌面端 App，无需配置复杂的环境变量。

2. **移动端与自托管后端探索**：
   - 移动端开发者可以进入 `app/` 目录，按照文档执行 `bash setup.sh ios`，通过 Flutter 编译运行手机端应用。
   - 后端开发者可以进入 `backend/` 目录，通过配置 `.env` 接入自己的 OpenAI API Key 和 Firebase，跑通基于 FastAPI 的自托管后端服务，实现数据的绝对私有化。

3. **硬件发烧友进阶**：
   - 如果资金充裕，可以在官网直接购买成品 Omi 硬件。
   - 动手能力强的硬核玩家，可以购买 Omi Glass Dev Kit (ESP32-S3)，参照官方文档自行烧录固件（`omi/` 或 `omiGlass/` 目录），甚至可以根据开源的硬件设计图，自己在嘉立创打板组装，打造独一无二的专属 AI 伴侣。
