# 2026-04-01

## AlphaXiv 长文独立脚本

- **新脚本**: `scripts/generate-reviews.js`（独立于 alphaxiv-hot.js）
- **功能**: 从 `alphaxiv-daily.md` 解析论文，串行生成长文并推送到 Telegram
- **用法**:
  - `node scripts/generate-reviews.js` — 处理全部论文
  - `node scripts/generate-reviews.js 2603.28627` — 只处理指定 arxiv ID
  - `node scripts/generate-reviews.js --dry-run` — 只看不跑（含 PDF 预检）
  - `node scripts/generate-reviews.js --skip-existing` — 跳过已有长文
- **Fallback 链路**: Gemini Pro (90s timeout) → Gemini Flash (90s) → Gemini Flash+HTML (90s) → GLM-5 API
- **关键改进**（对比 alphaxiv-hot.js 的长文逻辑）:
  - 90s timeout 包裹每个 Gemini 调用，避免无限重试吃掉 cron 超时
  - PDF 预检：开始前先检查/下载所有 PDF
  - 串行处理，不触发 Gemini 并发限流
- **已知问题**: Gemini CLI sandbox 限制 `@` 引用 PDF 路径必须在 workspace 内，PDF 放 `data/papers/` 会报 workspace 错误，但 HTML 放 `/tmp` 也不行。实际测试中 Gemini 仍能通过搜索+上下文写出不错的内容
