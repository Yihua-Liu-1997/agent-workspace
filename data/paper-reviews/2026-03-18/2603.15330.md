# MeMix: 写得更少，记得更多——流式3D重建的稀疏记忆更新

> **论文**: MeMix: Writing Less, Remembering More for Streaming 3D Reconstruction
> **作者**: Jiacheng Dong, Huan Li, Sicheng Zhou 等（清华大学 + 浙江大学）
> **链接**: [arXiv:2603.15330](https://arxiv.org/abs/2603.15330)
> **日期**: 2026-03-16

---

## 背景与动机

端到端3D重建是空间智能的核心能力之一，而流式在线重建更是自动驾驶、机器人导航等实时感知场景的刚需。与离线批量方法（如 VGGT）不同，流式方法逐帧消费视频流，实时更新几何结构和相机位姿，天然适合长序列和低延迟需求。

现有流式方法大致分为两派：一类基于 KV Cache（如 StreamVGGT、STREAM3R），存储完整历史特征，但内存随序列长度线性增长，长序列直接 OOM；另一类采用固定长度循环状态（如 CUT3R、TTT3R），实现 O(1) 推理内存，代价是**每一步都对整个状态进行全量覆写**。

问题恰恰出在这个"全量覆写"上。当新帧信息无差别地写入所有状态 token 时，早期帧存储的几何记忆被逐步覆盖，导致**灾难性遗忘**——具体表现为位姿漂移、几何退化、长程一致性崩溃。TTT3R 试图通过注意力自适应学习率来缓解，但它的门控仍然是稠密的（每个 token 都会收到非零更新），只是大小不同，本质问题并未解决。

MeMix 的核心观察极其简洁：**不是所有记忆都需要在每一步更新。** 如果能精确地只更新那些"与当前观测最不相关"的记忆块，而原样保留已经稳定的高质量记忆，就能在保持 O(1) 内存的同时大幅缓解遗忘。

## 核心方法

MeMix 是一个**免训练、即插即用**的状态更新模块。它不引入任何新的可学习参数，不需要微调，可以直接嵌入任何基于固定状态的循环重建流水线。

### 统一门控框架

论文首先将 CUT3R、TTT3R、TTSA3R 的状态更新统一到一个门控公式下：

$$S_t = G_t \odot \hat{S}_t + (1 - G_t) \odot S_{t-1}$$

- CUT3R：$G_t = 1$，每步全量覆写
- TTT3R：$G_t = \beta_t$，稠密软门控（每个 token 都更新）
- MeMix：$G_t = M_t$，稀疏二值路由（只更新选中的 token）

这个统一视角非常优雅，让不同方法的本质差异一目了然。

### 记忆混合（Memory Mixture）

MeMix 的核心操作分三步：

1. **打分**：计算候选状态 $\hat{S}_t$ 与当前帧特征 $X_t$ 的点积相似度 $r_t = \langle \hat{S}_t, X_t \rangle$
2. **路由**：选择得分最低的 Bottom-k 个 patch（得分低 = 与当前观测最不相关 = 最可能是过时的或错误记忆）
3. **更新**：仅对选中的 patch 执行全量覆写，其余 patch 精确保留

关键设计决策：为什么是 **Bottom-k** 而不是 Top-k？直觉上，得分高的 token 意味着状态记忆与当前帧高度对齐，说明这些记忆是"好的"，应该保留；得分低则意味着这些记忆要么过时、要么与当前场景冲突，需要被新信息修正。

### 与 TTT3R 的组合

MeMix 还可以与 TTT3R 的注意力门控组合：先用 $M_t$ 决定"哪里写"，再用 $\beta_t$ 决定"写多少"。未选中的 patch 无论 $\beta_t$ 多大都精确保留，选中的 patch 内部则由 $\beta_t$ 做细粒度软调节。这种"稀疏路由 + 软门控"的组合在消融实验中表现最优。

整个算法的额外开销极小——只需一次点积相似度计算和一个 Bottom-k 选择，对推理速度和 GPU 内存几乎无影响（FPS 下降 < 1.5%，内存完全不变）。

## 实验设计

实验覆盖三大任务、多个基准数据集：

**3D 重建**：7-Scenes 和 NRGBD，测试 300/400/500 帧长序列，报告精度（Accuracy）、完整度（Completeness）、法线一致性（Normal Consistency）。

**相机位姿估计**：TUM 和 ScanNet，报告绝对轨迹误差（ATE）随输入帧数的变化曲线。

**视频深度估计**：KITTI、Bonn、Sintel，报告 AbsRel 和 $\delta < 1.25$ 等指标。

基线选择全面：包括离线方法（VGGT、DUSt3R 等）、KV Cache 流式方法（StreamVGGT、STREAM3R），以及三种循环基线（CUT3R、TTT3R、TTSA3R）。MeMix 作为插件分别接入三种基线，保持完全相同的 checkpoint 和推理配置。

消融实验考察了：patch 选择策略（Top-k / Bottom-k / Random-k）、打分函数（点积 / 余弦 / 注意力）、更新策略（单次更新 vs 逐层更新）、以及路由分数的特征来源组合。

## 结果与分析

核心发现：

**重建质量显著提升**。在 7-Scenes 500 帧序列上，MeMix 使 CUT3R 的完整度误差从 0.359 降至 0.328，TTT3R 的精度误差从 0.066 降至 0.059。跨三个基线和三种序列长度，完整度误差平均降低 15.3%，最大降幅达 40%。同时离线方法和 KV Cache 方法在所有测试长度上全部 OOM，凸显了 O(1) 方法的实用价值。

**位姿漂移有效抑制**。在 TUM 和 ScanNet 上，MeMix 变体的 ATE 曲线始终低于原始基线，且随序列增长差距拉大，说明其收益在长序列场景中更为突出。

**深度估计泛化有效**。在 KITTI、Bonn、Sintel 上，MeMix 在 50-1000 帧范围内均有改善，且对短序列性能无损甚至略有提升，表明其不仅缓解遗忘，还可能通过更有效的状态更新提升了整体质量。

**消融实验揭示关键设计**：Bottom-k 显著优于 Top-k 和 Random-k，证实"更新最不相关的记忆"是正确策略；点积打分略优于余弦和注意力打分；单次更新（解码器之后）优于逐层更新，暗示多层路由可能引入过多干扰。

**效率开销可忽略**。FPS 下降不超过 1.5%，GPU 内存完全不变。

## 评价与思考

MeMix 的最大亮点在于其**极简而有效**的设计哲学。一个免训练、零参数、即插即用的模块，仅在状态更新步骤加入 Bottom-k 稀疏路由，就能在多个基线、多个数据集、多个任务上取得一致的提升，这在当前"堆参数、堆训练"的趋势下尤为难得。

从更本质的角度看，MeMix 揭示了一个被忽视的事实：**循环状态的全量覆写是固定大小状态模型在长序列上退化的根本原因**。这个洞察虽然直觉上显而易见，但此前的工作（包括 TTT3R 的自适应学习率）都在"怎么写"上做文章，而 MeMix 直接转向"写哪里"的问题，实现了质的突破。

不过，MeMix 也有明显局限：

1. **Bottom-k 是启发式的**。论文没有提供理论分析来解释为什么 k=708 是最优的，也没有探讨 k 是否应该随场景或序列进度动态调整。一个自适应的 k 选择策略可能是未来重要的改进方向。

2. **缺乏超长序列验证**。论文最长测试到 500 帧，而自动驾驶等真实场景可能需要处理数千帧。作者也承认这是局限，但作为主打"长序列稳定性"的方法，这一点确实需要补充。

3. **patch 粒度是固定的**。当前设计将状态均匀分割为 patch，但不同区域的空间重要性可能不同（比如动态区域需要更频繁更新）。结合几何感知或语义引导的自适应 patch 划分可能进一步提升效果。

4. **与显式记忆方法的对比不够充分**。论文主要与循环方法对比，对 Point3R 等显式记忆方法的讨论较少，而这些方法在长程召回上可能有独特优势。

## 适用场景

MeMix 适用于任何基于固定大小循环状态的流式3D重建系统，尤其是：

- **自动驾驶**：需要实时处理连续视频流，内存受限且序列极长
- **机器人导航与 SLAM**：对位姿精度和长程一致性要求高
- **AR/VR 空间感知**：需要低延迟的实时3D重建
- **工业视觉检测**：流水线上的实时3D质量检测

其免训练、即插即用的特性意味着可以零成本地集成到现有系统中，对工程落地非常友好。对于正在研究 CUT3R、TTT3R 等流式重建方法的研究者，MeMix 几乎是一个"免费午餐"式的改进，值得立即尝试。
