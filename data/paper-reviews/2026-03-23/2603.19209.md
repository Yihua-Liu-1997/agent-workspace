# Do VLMs Need Vision Transformers? Evaluating State Space Models as Vision Encoders
视觉语言模型需要视觉 Transformer 吗？评估状态空间模型作为视觉编码器

## 论文信息
- **作者**：Shang-Jui Ray Kuo, Paola Cascante-Bonilla
- **机构**：Stony Brook University（石溪大学）
- **arXiv 链接**：https://arxiv.org/abs/2603.19209

## 背景与动机
近年来，大型视觉-语言模型（VLMs）通常采用模块化设计：一个预训练的视觉编码器产生视觉 token，一个轻量级的连接器（connector）将它们映射到大型语言模型（LLM）的嵌入空间中，然后整个系统经过指令微调以实现开放端生成。

在当前的 VLM 训练配置中，绝大多数模型仍然依赖基于 Transformer 的架构（如 ViT 系列）作为视觉主干网络。这引发了一个关键问题：VLM 是否必须使用 ViT？其他架构是否能提供更优的视觉表示？

同时，从图像中提取空间维度的证据（grounding evidence）在固定的多模态 token 预算下面临巨大挑战。为了捕捉细节，通常需要增加图像分辨率或视觉 token 数量，但这会导致视觉编码器和 LLM 端的计算和内存成本急剧上升。状态空间模型（State Space Models, SSMs）近期在密集预测等视觉任务中展现了强大的性能，由于其能通过多向扫描在 2D 网格上保留细粒度的空间信息，因此具有成为优秀 VLM 视觉编码器的潜力。但在受控的训练配方和相同的视觉-语言接口下，SSM 在 VLM 中的表现还缺乏系统的评估。

## 核心方法
本研究没有提出全新的模型架构，而是通过**控制变量法（Controlled Evaluation）**对不同视觉主干在 VLM 中的表现进行了深度剖析。主要方法包括：

1. **统一模型框架**：采用典型的 LLaVA 风格架构，固定语言模型（Vicuna-7B）和训练配方（包括数据、优化器等），仅替换冻结的视觉编码器，以隔离单纯的架构效应对 VLM 性能的影响。
2. **多架构对比**：横向对比了标准 ViT、混合架构（MaxViT、MambaVision）以及纯 SSM 架构（VMamba）。
3. **预训练目标探索**：除了对比传统的 ImageNet-1K 分类预训练模型，还引入了针对密集预测任务（目标检测、语义分割）微调后的视觉编码器（如 ViTDet、DeiT 以及密集任务版的 VMamba 等），以评估密集型预训练目标对下游 VQA 和定位任务的影响。
4. **诊断与稳定策略**：针对某些检测预训练模型在大分辨率下出现的“定位崩溃（localization collapse）”问题，提出从“传输瓶颈（Transmission bottleneck）”和“利用瓶颈（Utilization bottleneck）”两个角度进行诊断，并给出了增加连接器容量（从 2 层 MLP 改为 3 层）和调整输入几何形状（例如强制使用 512x512 正方形输入）的稳定化策略。

## 实验设计
- **训练数据**：使用了 LLaVA 提供的 665K 多模态指令微调数据集进行微调。
- **评估基准**：全面涵盖了 VQA（如 VQA-v2, GQA, VizWiz, TextVQA, POPE, TallyQA）和定位/基础任务（RefCOCO, RefCOCO+, RefCOCOg, OCID-Ref）。
- **实验设置分类**：
  - **受控的 IN1K/224 匹配替换**：在严格相同的输入分辨率（224x224）和视觉 token 数量（L=196）下横向对比不同架构的表现。
  - **密集目标预训练模型对比**：使用更高的图像分辨率（根据预训练时的尺寸）评估那些经过检测和分割微调的检查点。

## 结果与分析
1. **SSM（VMamba）在严格控制下表现优异**：在 IN1K 初始化的 matched 替换实验中，VMamba（尤其是 Tiny 和 Small 版本）在各种定位（Grounding）基准上一致领先，且在整体综合 VQA 表现上也是最强的。这归功于 VMamba 的 SS2D 设计内在地通过多向聚合保留了 2D 空间结构，而 ViT 的自注意力机制在缺乏显式定位目标时，容易使位置编码包含的空间信息在深层退化。
2. **密集任务预训练（Dense Objectives）有显著增益**：使用目标检测或语义分割进行适配的预训练视觉编码器通常能够提升 VQA 和定位的性能。这在原本空间归纳偏置较弱的 ViT 家族中效果尤为显著（例如 ViTDet 对比普通 ViT）。
3. **ImageNet 准确率和模型规模具有欺骗性**：实验揭示了一个重要的反直觉现象：视觉编码器在 ImageNet 上的更高分类精度，以及单纯增大模型参数量，并不能可靠地转化为更好的下游 VLM 性能，有时甚至会倒退。这可能是由于“分类目标过拟合（objective overfitting）”，即更大的模型在分类任务上过度拟合，倾向于丢弃细粒度空间信息而仅仅保留识别显著物体的全局语义。
4. **发现并解决“定位崩溃（Localization Collapse）”**：一些高分辨率检测微调的模型（如 ViTDet-L/H 和 VMamba-T/B）在接入 VLM 且使用长宽比非正方形的高分辨率输入时，定位能力突然剧烈丧失。实验分析表明，这本质上是视觉-语言的接口兼容问题，并非特征提取失败。通过简单地增加 MLP 连接器的深度（提高信息传输容量）以及使用方形输入分辨率（改善 LLM 对空间线索的解释和利用），可以有效消除崩溃现象，并大幅提升各项任务的表现。

## 评价与思考
这篇论文的切入点非常敏锐且具有极高的工程实用价值。当前，无论开源还是闭源的 VLM 社区，几乎都在“无脑”堆叠基于 ViT 的视觉特征（如 CLIP、SigLIP 加上 DINOv2），并通过拼图等方式提高分辨率，而本文则通过严谨的消融实验证明了路线的多样性与可能性。

1. **空间信息的保存是 VLM 的核心瓶颈之一**：论文深刻地指出了 SSM 架构的多向扫描机制自带强大的空间结构保持能力。这使得 SSM 在资源受限（相同的 token 数量）时，能够提供比 ViT 更加凝聚、清晰的局部特征。这对抑制模型幻觉、提高细粒度问答精度和强化视觉定位至关重要。
2. **反直觉的结论引发深思**：ImageNet 准确度越高的视觉主干，在 VLM 里的效果反而可能变差。这一发现向整个多模态预训练领域发出了强烈的信号——为单模态图像分类高度优化的特征，在跨模态生成及空间推理任务中可能是一种“有损压缩”。我们需要专门为 VLM 设计对空间更敏感的稠密预训练目标，而不仅仅是打榜 ImageNet。
3. **工程视角的启发**：“定位崩溃”现象的诊断和解决非常精彩。很多时候当模型在下游任务表现不佳时，研究者习惯于责怪训练数据不足或者视觉编码能力不够，但本文证明了简单的接口协议调整（增大一点 Connector 的容量，或是将长宽比固定为正方形以降低 LLM 理解坐标位置的难度）就能挽救一个看似失败的超大模型。这也暗示当前 VLM 将 2D 图像暴力拉平成 1D 序列喂给 LLM 的方式本身就存在一定的脆弱性。

总而言之，本文为下一代端侧和高效 VLM 的设计提供了强有力的替代思路：以 SSM 为代表的轻量级、空间敏感型编码器，在特定参数量级下，完全有能力甚至能更好地替代传统笨重的 Vision Transformers。

## 适用场景
- **端侧/轻量化多模态应用**：SSM 本身具有线性复杂度的推理高效性，并且在较少 Token（例如 196 个）下展现出了极其优秀的细粒度定位能力。这使得它非常适合计算资源、内存和功耗受限的移动端设备或机器人终端系统。
- **密集视觉理解与精准定位任务**：对于需要精准指出物体坐标、读取密集文本、分析复杂场景图的场景（如具身智能的抓取动作、医疗图像病灶分析、复杂的网页及文档结构理解等），使用密集预训练结合 SSM 作为主干网络，能提供极具优势的精细空间特征。
- **高效视频-语言模型（Video-LMs）的演进**：考虑到视频本质上是更长的 token 序列和更复杂的时空结构，纯注意力机制在处理视频的长上下文时会面临 $O(N^2)$ 的内存爆炸。既然 SSM 在 2D 图像上能有效保留空间关联，若将其扩展至 3D (时空维度)，有望在显著降低视频序列计算复杂度的同时保留高保真的时空运动轨迹特征。
