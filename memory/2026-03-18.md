## 多 Agent 方案讨论

### 背景
- 用户女朋友是教师，想用 bot 辅助写教案
- 用户明确想创建第二个 agent（不接受纯 systemPrompt 软限制方案）

### 调研结果
- OpenClaw 支持多 agent：`agents.list` 定义多个 agent，`bindings[]` 按群/频道路由
- **同一个 bot 可以有多个 agent**，不需要创建新 bot
- 配置示例：
  ```json5
  agents: {
    list: [
      { id: "main", /* 现有配置不动 */ },
      { id: "teacher", model: { primary: "zai/glm-5-turbo" },
        workspace: "~/.openclaw/workspace-teacher" }
    ]
  }
  ```
- 通过 bindings 把女朋友的群路由到 teacher agent

### 重大发现：OpenClaw 支持按 agent 限制工具！
- 之前误以为不支持，后来在 `/docs/zh-CN/tools/multi-agent-sandbox-tools.md` 找到完整文档
- **`agents.list[].tools.allow` / `agents.list[].tools.deny`** 可以按 agent 设置工具白名单/黑名单
- 还支持工具组简写：`group:runtime`、`group:fs`、`group:ui` 等
- 还支持 per-agent Docker 沙箱：`agents.list[].sandbox`
- 还支持 per-agent exec-approvals 白名单

### 最终方案讨论
- 用户想让女朋友体验"AI 真的在帮我干活"的感觉
- 讨论了三个方案：
  1. Docker 沙箱（本地容器，简单但她在自己电脑上看不到）
  2. OpenClaw Node（远程机器，她能看到但需要装 openclaw）
  3. SSH + exec 白名单（太笨重，放弃）
- **选定方案 B：OpenClaw Node** — 她的电脑装 openclaw node，连回 Gateway，在她电脑上干活
- 网络问题：Clash TUN 模式可能影响 Node 连接
  - 直连方案：Gateway 改绑 `0.0.0.0`，路由器端口转发，入站连接一般不经 TUN
  - Tailscale 方案：需要 Clash 配 `route-exclude-address: 100.64.0.0/10`
- Gateway 当前绑定 `127.0.0.1:18789`，需要改 `0.0.0.0`
- WSL2 IP 是 Clash TUN 虚拟 IP（198.18.0.1），需要 Windows 公网 IP
