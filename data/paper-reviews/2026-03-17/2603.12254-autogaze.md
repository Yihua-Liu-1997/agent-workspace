# AutoGaze: 在注意力之前的选择性注视

**论文标题**: Attend Before Attention: Efficient and Scalable Video Understanding via Autoregressive Gazing  
**arXiv ID**: 2603.12254  
**发表时间**: 2026年3月  
**作者机构**: UC Berkeley, MIT, Clarifai, NVIDIA

---

## 一、背景与动机

人类观看动态场景时，眼睛会快速移动，聚焦于运动物体并捕捉精细细节，同时跳过静态背景。这种选择性注视机制使我们能够实时处理高帧率、高分辨率的视觉流。然而，现代视频理解模型——尤其是多模态大语言模型（MLLMs）——仍然平等地处理每一帧的每一个像素，在存在大量时空冗余的视频数据上浪费了巨额计算资源。

现有的视频冗余削减方法主要在 LLM 层面进行 token 剪枝，而视觉编码器（ViT）仍然处理全量像素，形成了严重的效率瓶颈，阻碍了模型向更长、更高分辨率视频的扩展。此外，现有方法要么依赖启发式规则（如注意力分数）导致性能次优，要么涉及繁重的搜索和推理过程，增加了额外开销。

针对这些挑战，来自 UC Berkeley、MIT、Clarifai 和 NVIDIA 的联合团队提出了 **AutoGaze**——一个仅 300 万参数的轻量级模块，在 ViT 处理之前智能地选择信息丰富的图像块（patch），去除冗余内容。

## 二、核心方法

### 2.1 模型设计

AutoGaze 采用了一个简洁的架构：卷积编码器 + 自回归 Transformer 解码器。给定一段视频，AutoGaze 交替执行帧编码和 patch 注视：

1. **自回归注视机制**：首先编码第一帧，然后自回归地解码 patch 索引。这与 LLM 的解码过程类似，但词汇表只包含 patch 索引而非单词。后续帧的注视会参考之前帧的特征和已注视的 patch 索引，从而避免选择冗余区域。

2. **自动停止机制**：模型配有一个预测头，在解码每个 patch 时预测当前重建损失。一旦预测损失低于用户设定的阈值，该帧的注视自动停止，转向下一帧。

3. **多尺度注视**：解码器词汇表包含多个尺度的 patch，使模型能够根据区域细节程度选择不同粒度——细节丰富的区域使用高分辨率，简单区域使用低分辨率。

4. **多 token 预测**：采用多头同时输出多个 patch 索引和重建损失，加速注视过程。

### 2.2 训练流水线

AutoGaze 采用两阶段训练：

**阶段一：Next-Token Prediction 预训练**

在 800K 视频数据集上收集近优注视序列（通过贪婪搜索获得），然后使用标准的下一 token 预测损失训练模型。同时监督重建损失预测头，使模型学会在任意注视长度下做出合理选择。

**阶段二：强化学习后训练**

由于预训练数据只包含次优注视，团队使用 GRPO 算法进行 RL 后训练，以重建损失作为奖励信号，发现更优的注视序列。

## 三、实验设计

### 3.1 HLVid 基准测试

团队注意到现有长视频基准测试缺乏高分辨率视频，因此提出了 **HLVid**——首个高分辨率长视频问答基准。它包含 268 个 QA 对，视频时长长达 5 分钟，分辨率达 4K，问题设计需要 1K-2K 分辨率的视觉感知能力才能回答。

### 3.2 实验设置

- **ViT**: SigLIP2-SO400M
- **MLLM**: NVILA-8B-Video
- **评估基准**: ActivityNetQA, EgoSchema, VideoMME, MLVU, LVBench, HLVid

## 四、结果与分析

### 4.1 注视行为分析

实验表明，AutoGaze 的注视决策具有高度原则性：

1. **关注运动区域**：在 FlyingChairs 数据集上测试，AutoGaze 在所有尺度上都更频繁地选择高光流（高运动）的 patch。

2. **尺度与细节关联**：通过拉普拉斯方差衡量 patch 细节程度，发现 AutoGaze 在细粒度尺度上选择更详细的 patch，且对更详细的区域使用更精细的尺度（ρ=0.12, p<0.001）。

3. **泛化能力强**：在未见过的场景（CCTV 监控、机器人操作、物体替换视频）和未见过的视觉风格（风格迁移后的视频）上，AutoGaze 仍能稳定追踪变化区域。

### 4.2 效率提升

对于 30 FPS、4K 分辨率的视频，AutoGaze 仅需约 1% 的 patch 即可达到 0.7 的重建损失阈值。这带来了：

- **ViT 加速**: 最高 19×
- **MLLM 加速**: 最高 10×

### 4.3 MLLM 扩展性能

借助 AutoGaze 的效率优势，团队将 NVILA-8B-Video 扩展至处理 1K 帧、4K 分辨率的视频：

- **VideoMME**: 67.0%（与 Qwen2.5-VL 等强模型竞争）
- **HLVid**: 52.6%（相比基线 42.5% 提升 10.1%，超越前最佳 MLLM 4.5%）

### 4.4 对比基线

与启发式注视方法（如均匀采样、随机采样、光流采样）相比，AutoGaze 能用更少的 patch 达到相同的重建质量。与现有 token 剪枝方法（LongVU, FastVID, VideoChat-Flash）相比，AutoGaze 的核心优势在于**在 ViT 之前**去除冗余，而非仅在 LLM 层面剪枝。

## 五、评价与思考

### 5.1 优势

1. **理念优雅**：模拟人类视觉的选择性注视机制，在计算发生之前就去除冗余，而非事后补救。

2. **轻量高效**：仅 300 万参数，几乎不增加推理开销，却能带来 10-20 倍的加速。

3. **自适应性**：重建损失阈值可调，用户可根据需求平衡质量与速度。

4. **泛化能力**：无需针对新分辨率/时长重新训练，可直接应用于 4K、1K 帧视频。

### 5.2 潜在局限

1. **依赖重建质量**：注视决策以重建损失为优化目标，但最小重建损失不一定对应最佳下游任务性能。

2. **训练数据成本**：需要预先收集近优注视序列，虽然可用贪婪搜索自动化，但仍有一定成本。

3. **任务无关性**：注视选择与具体任务解耦，可能导致任务无关区域被过度关注。

## 六、适用场景

1. **长视频理解**：需要处理数分钟甚至更长视频的场景，如电影/剧集分析、监控视频检索。

2. **高分辨率视频分析**：4K/8K 视频的精细识别任务，如医疗影像、卫星图像、工业检测。

3. **资源受限环境**：边缘设备、移动端视频处理，需要高效推理的场景。

4. **实时视频流处理**：需要低延迟的视频分析应用，如自动驾驶、机器人视觉。

---

## 结语

AutoGaze 以"在注意力之前进行选择"的简洁理念，为视频理解模型的可扩展性提供了新的解决方案。其轻量级设计、自适应注视机制和出色的泛化能力，使其有望成为未来视频 MLLM 的标准组件。HLVid 基准的提出也填补了高分辨率长视频评估的空白，为社区提供了重要的测试工具。
