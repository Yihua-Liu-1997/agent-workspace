# Agentic-MME: What Agentic Capability Really Brings to Multimodal Intelligence?
（Agentic-MME：智能体能力究竟能为多模态智能带来什么？）

## 论文信息

## 背景与动机
当前的多模态大语言模型（MLLMs）正处于从“被动观察者”向“主动智能体（Active Agents）”快速进化的阶段。为了解决真实世界中的复杂问题，这些模型不仅需要看，还需要主动地通过工具干预和拓展信息。这种**多模态智能体能力**可被拆分为两个核心维度：
1. **视觉拓展（Visual Expansion）**：让模型能够“用图像思考”，通过裁剪、旋转、增强等视觉操作处理输入图像，挖掘出被遮挡或模糊的潜在视觉线索。
2. **知识拓展（Knowledge Expansion）**：让模型超越其自身的参数化记忆，通过开放的网页搜索去验证视觉之外的现实事实并解决歧义。

虽然这一主动范式极具潜力，但现有的基准测试存在三个致命缺陷：
1. **缺乏工具集成的灵活性和全面性**：现有测试往往将视觉工具和网页搜索割裂开来，缺乏允许智能体在异构工具中无缝切换和调用的统一框架。
2. **缺乏视觉与知识协同的探索**：缺乏真实的测试场景来检验这两项能力是否能够协同工作（Synergy），而正是这种协同才体现了多模态智能体的真正价值。
3. **缺乏严谨的过程验证**：以往的评估过度依赖“最终答案”的正确性，无法判定模型是否真正调用了工具、是否正确提取了特征，或者是否在无效的试错中浪费了算力。

为了弥补这些空白，作者提出了 **Agentic-MME**——一个专注于全面评估多模态智能体能力且具备详细过程验证的新基准。

## 核心方法
Agentic-MME 构建了一个集成了视觉工具与开放式网页搜索的统一工作流，并包含418个真实任务（跨越6个领域），任务设计依据交互复杂度分为三个递进难度：
- **Level 1（视觉拓展为主）**：孤立的单步视觉操作（如仅需一次裁剪即可发现线索），测试基础的感知-行动循环。
- **Level 2（短序列多步工作流）**：要求模型进行视觉操作后再结合简单的外部知识检索，通常能在三步左右的线性工具链内解决。
- **Level 3（深度协同）**：现实且极具挑战的场景。模型无法通过线性操作解决，必须交替、迭代地使用视觉和搜索工具。例如：先裁剪出一个模糊Logo，通过搜索匹配候选品牌，再结合检索到的事实对处理后的图像进行假设验证。

**过程验证（Process-Verified Evaluation）的双轴系统**：
为了打破“只看结果”的黑盒，Agentic-MME 提供了超过 2,000 个细粒度的人类标注检查点（Checkpoints）：
- **S-axis（策略轴）**：审计外部知识拓展的过程，评估每一步搜索的关键词、人类验证的参考URL以及检索到的中间答案是否正确。
- **V-axis（视觉轴）**：审计视觉拓展的中间状态，甚至被进一步拆分为 $V_{tool}$（是否有意图调用正确的视觉工具）和 $V_{true}$（中间生成的视觉图像中是否真实包含了所需证据）。

**统一评估框架与效率指标**：
- 支持两种异构的交互接口：沙盒环境中的 Python 代码执行（Gen 模式，利用 AST 追踪视觉操作）以及结构化函数调用 API（Atm 模式）。
- 引入了**Overthink（过度思考）** 指标：通过将智能体的执行步骤与人类的最小操作轨迹对比，量化并惩罚冗余的、毫无意义的循环调用和试错。

## 实验设计
研究团队选取了目前最具代表性的开源与闭源多模态模型进行评估：
- **闭源模型**：Gemini 3 Pro-preview, Gemini 3 Flash, Qwen 3.5-plus, Kimi-k2.5, GPT 5.2, GPT 5-mini 等。
- **开源模型**：Qwen 3 VL系列 (8B, 32B, 235B), Deepeyesv2, Thyme-rl 等。
- 实验评估时，所有的模型分别在 **Code（代码生成，Gen）** 模式和 **Atomic（结构化API，Atm）** 模式下进行测试。在验证中间过程时，借用了基于 LLM-as-a-Judge 的机制（以 GPT-4o-mini 作为主要裁判）对比人类的标注真值。

## 结果与分析
实验结果揭示了几个极其重要的发现：
1. **模型与人类存在巨大鸿沟，特别是在 Level-3 任务中遭遇滑铁卢**：
   人类求解的整体准确率高达 93.8%，在最高难度 Level-3 中依然有 82.3%。表现最好的模型 Gemini 3 Pro (Atm模式) 整体准确率为 56.3%，但在 Level-3 骤降至 23.0%（尽管工具让其在 L3 的能力从纯感知的 7.5% 提升到了 23%，但仍有极大的提升空间）。
2. **开源模型在搜索规划方面大幅落后**：
   大部分开源模型在 S-axis（策略检索）上的得分极低。例如 Thyme-rl 和 Deepeyesv2 在 S-axis 的得分甚至不到 5%，这意味着它们虽然学会了调用本地视觉工具，但缺乏可靠的多步问题解决与信息检索规划能力。
3. **结构化 API 优于 代码生成，但代码模式有潜在优势**：
   几乎所有模型在 Atm（结构化调用）模式下表现均好于 Gen（写 Python 代码）模式。代码生成要求模型掌握各种包引入、文件读写、错误处理等，认知负荷过重，频繁发生语法等低级错误；但代码模式保留了极高的操作自由度。
4. **“眼高手低”与“过度思考”现象严重**：
   许多模型非常“热衷”于调用工具（$V_{tool}$ 得分高），但经常裁错地方或加错滤镜，导致实际效果极差（$V_{true}$ 极低）。此外，像 GPT 5-mini 这样的模型平均每个任务调用工具高达 12 次以上，陷入了极度低效的过度试错。

## 评价与思考
《Agentic-MME》这篇研究切中了当前大模型领域“从语言大模型到智能体（Agent）”转型过程中的关键痛点：**如何衡量多模态Agent的“真实工具使用过程”。**

1. **打破结果导向的黑盒，提升了评测透明度**：
   以往的测试大多只看最终结果，这就导致了难以分辨模型究竟是真正“想透了”问题，还是瞎猫碰到死耗子。Agentic-MME 通过耗费极大心血的人工标注，引入了 S/V-axis 的过程验证机制，在当前大模型的自动化评测中具有里程碑式的意义。
2. **深度定义了知识与视觉的“协同（Synergy）”**：
   文章在 Level 3 任务中揭示了人类解决复杂问题的范式：“看图发现模糊线索 -> 搜索求证 -> 再次反观图片确认”。这种跨模态协同规划正是当前 AI 的软肋。研究有力证明了“视觉处理能力”和“外部知识检索能力”不是孤立的技能包，必须在复杂的动态环境中交织进行。
3. **揭示了大模型落地的瓶颈与优化方向**：
   测试暴露了当前前沿 AI 在“认知”与“执行”上的割裂——AI 知道应该去裁剪，但却精准地裁剪了无用的背景。这暗示未来的 Agent 训练不能仅依赖最终答案层面的监督学习，而必须引入更为细化的基于中间过程的强化学习（Process-supervised RL/GRPO），并在训练中对无效的“过度思考（Overthink）”施加惩罚，以提升执行的准确性和效能。

## 适用场景
基于本论文提出的理论体系与基准测试，其评估方法和任务设计对以下应用场景具有直接指导价值：
1. **下一代多模态 Agent 研发与能力诊断**：帮助顶尖 AI 实验室（如 OpenAI、Google、DeepSeek 等）诊断其模型在推理链条和工具调用中的薄弱环节。
2. **RPA与图形界面自动化（UI Agent）**：如今的手机/电脑终端助手需要精准“阅读”屏幕截图，识别模糊的验证码或图标，然后利用网页搜索进行自动决策，这一过程完美契合了视觉与知识协同的能力需求。
3. **智能数据调查系统**：对于需要结合图片物证、卫星地图并关联海量互联网信息进行情报收集、反欺诈分析、事实核查的系统，该研究提供了一个可量化和优化的健壮推理工作流参考。
