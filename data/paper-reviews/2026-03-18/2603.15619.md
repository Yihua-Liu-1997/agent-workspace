# MoDA: Mixture-of-Depths Attention — 深度感知的统一注意力机制

> **论文**: Mixture-of-Depths Attention  
> **作者**: Lianghui Zhu, Yuxin Fang, Bencheng Liao 等（华中科技大学 & 字节跳动 Seed）  
> **arXiv**: 2603.15619, 2026-03-16  
> **代码**: https://github.com/hustvl/MoDA

---

## 一、背景与动机

大语言模型（LLM）的进步沿着四个维度展开：上下文长度、训练数据、模型宽度和模型深度。然而，在这四条路中，**深度（depth）** 始终是最被"冷落"的一个。

原因很现实：宽度扩展容易优化、系统效率高，而深度扩展面临着两大顽疾——**优化困难**和**信息稀释**。标准 Transformer 通过残差连接（ResNet-style）在层与层之间传递隐藏状态，但残差连接本质上是一个"不断叠加"的过程：每层都在同一个固定大小的张量上反复读写。浅层形成的有用特征，在经过几十层叠加后会被逐渐"稀释"，深层网络难以从残差流中精确恢复这些早期信息。

为了缓解信息稀释，研究者尝试过 DenseNet 风格的跨层密集连接，保留所有中间层的表示。但代价是惨重的——参数量以 O(L²D²) 的速度增长，在大模型规模下根本不可行。

本文提出了一个优雅的问题：**能不能像注意力机制解决序列建模那样，用数据依赖的动态混合来解决深度建模中的信息传递问题？** 这正是 Mixture-of-Depths Attention（MoDA）的核心思想——让每一层的注意力头既能关注当前层的序列 KV 对，也能动态地从所有前置层的深度 KV 中检索信息，并在一个统一的 softmax 中完成。

## 二、核心方法

MoDA 的设计可以用三个层次来理解：

### 2.1 设计空间分析：Read → Operate → Write

作者提出了一个统一的分析框架，将每一层 Transformer 视为三个步骤：

- **Depth Residual**（标准残差）：read 是恒等映射，write 是加法。简单但会压缩历史信息。
- **Depth Dense**（密集连接）：read 拼接所有历史层表示，write 也是拼接。无损但参数爆炸。
- **Depth Attention**（本文中间方案）：用注意力在历史层的 KV 上做数据依赖的读取。参数量降至 O(LD²)，比 DenseNet 低一个数量级。
- **MoDA**（最终方案）：将序列注意力和深度注意力融合进一个统一的 softmax 操作。

### 2.2 MoDA 的具体机制

在 MoDA 中，每个注意力头的 Query 不仅与当前层的 Sequence KV 计算注意力，还与**同一 token 位置在所有前置层的 Depth KV** 计算注意力。所有注意力分数在一个共享的 softmax 中归一化，形成一个统一的"序列 + 深度"注意力空间。

一个关键设计是 FFN 层的深度 KV 投影：由于 Transformer 每层包含 Attention 和 FFN 两个子层，只保留 Attention 层的 KV 会丢失 FFN 处理后的信息。MoDA 为 FFN 层引入了轻量的 KV 投影，将 FFN 的输入投影为深度 KV 对，供后续层使用。实验表明这带来了显著的额外增益。

### 2.3 硬件感知的高效实现

MoDA 的理论很优雅，但朴素实现会因非连续内存访问导致 GPU 利用率极低。作者开发了三个关键优化：

1. **Flash 兼容的深度 KV 布局**：将深度缓存沿单一轴展平，使每个 token 的 L 个历史状态连续存储。相比 PyTorch 朴素实现实现了 **162 倍加速**。

2. **分块感知的深度 KV 布局**：将查询按块分组，每个块只访问对应的深度 KV 范围，将深度利用率从 1/T 提升到 1/C，额外减少 52% 运行时间。

3. **分组感知索引**：利用 GQA 中 G 个相邻查询行共享同一 base-time 索引的特性，进一步减少重复计算，带来 4.3 倍额外加速。

三者叠加后，MoDA 在 A100 上实现了 FlashAttention-2 **97.3%** 的效率（序列长度 64K 时仅慢 2.73%），同时保持了数值精度在允许范围内。

### 2.4 复杂度分析

MoDA 在参数量上达到 O(LD²/G)，是三种深度感知方案中最优的（得益于复用序列注意力的 Query 投影）。FLOPs 和缓存均为 O(L²D) 和 O(LD/G)，与 Depth Attention 相当，远优于 Depth Dense 的 O(L²D²)。

## 三、实验设计

实验在 OLMo2 的 400B token 训练方案上进行，覆盖两个模型规模（700M 和 1.5B），主要对比基线为 OLMo2 原始模型。

**MoDA 变体消融**（700M）：
- 是否使用 Depth KV
- 是否为 FFN 层添加额外 KV 投影
- 是否为 Attention 层添加额外 KV 投影

**规模扩展实验**：从 700M 扩展到 1.5B，验证增益的稳定性。

**层深分析**：在 24 层和 48 层的小模型上，分别测试 pre-norm 和 post-norm 配置。

**注意力可视化**：展示 MoDA 如何在不同层和头之间分配序列与深度的注意力权重。

**效率消融**：逐步启用三个硬件优化，量化每个优化的贡献。

**评估覆盖**：10 个下游 benchmark（PiQA、HellaSwag、WinoGrande、MMLU 等）+ 10 个领域的验证困惑度。

## 四、结果与分析

### 4.1 下游任务表现

在 1.5B 模型上，MoDA 在 10 个下游任务上的平均性能从 62.28 提升到 64.39（**+2.11%**），且 10 个任务全部获得提升。亮点包括：
- WinoGrande: +2.37%
- ARC-Challenge: +4.35%
- BoolQA: +3.73%
- MMLU: +1.86%
- COPA: +4.00%

增益覆盖常识推理、因果判断、科学知识和广泛知识，表明 MoDA 带来的深度信息检索能力具有普遍价值。

### 4.2 困惑度改善

1.5B 模型在 10 个领域的平均验证 PPL 从 13.67 降至 13.47，10 个领域**全部改善**，尤其在 Reddit（-0.36）、ICE（-0.29）和 Wiki-text（-0.25）上改善显著。

### 4.3 变体消融的关键发现

- **复用序列 KV 作为深度 KV**：几乎零成本（仅 0.12% FLOPs 增加），却带来 0.41 训练 PPL 改善。
- **FFN 层 KV 投影**：额外 5.4% 参数增加，换来 0.27 C4 Val PPL 和 0.77 下游指标的额外改善。
- **额外 Attention KV 投影**：边际收益递减（仅 +0.10 下游），但参数增加 5.2%——不值得。

### 4.4 Post-norm 与 Pre-norm

一个意外发现：MoDA 与 **post-norm** 配合效果显著优于 pre-norm。在 48 层模型上，post-norm + MoDA 的 FineWeb-Edu Val Loss 降幅（0.0409）远大于 pre-norm（0.0041）。这与近期"post-norm 回归"的研究趋势一致，MoDA 的深度信息通道可能恰好缓解了 post-norm 在深层网络中的优化不稳定性。

### 4.5 注意力可视化

可视化揭示了两个有趣现象：
1. 中间层和深层持续将显著注意力质量分配给深度 KV 块，而非仅依赖序列注意力。
2. MoDA 展现出与传统 **attention sink** 不同的模式——概率质量更均匀地分布在有信息的序列和深度位置上，而非集中坍缩到少数固定 sink 位置。

## 五、评价与思考

### 亮点

1. **思路自然且优雅**：将"注意力解决序列混合"的原则推广到"深度混合"，概念上非常统一，读起来有种"为什么之前没人做"的感觉。
2. **工程完成度高**：从理论到 Triton kernel 全栈实现，效率优化层层递进（162× → 再 2× → 再 4.3×），最终达到 FlashAttention-2 的 97.3%——这不是一个只停留在纸面上的 idea。
3. **实验扎实**：多规模验证、多维度评估、详尽的消融实验，加上可视化分析，形成了完整的证据链。
4. **开源承诺**：代码即将开源，对社区友好。

### 局限与思考

1. **模型规模仍偏小**：700M 和 1.5B 在当今 LLM 时代只能算中小模型。MoDA 在 7B+ 规模上的表现尚待验证，尤其是深度 KV 缓存的显存开销在大模型上会显著增加。
2. **KV Cache 推理开销**：MoDA 的推理缓存从标准 FlashAttention 的 O(TD) 增长到 O(TLD/G)。对于 L=64、G=8 的配置，缓存增加 8 倍，这对长文本推理的显存压力不容忽视。作者在 Discussion 中提到了 bounded depth-KV slot caching 的方向，但目前尚未实现。
3. **与 MoE 的关系未探讨**：MoDA 本质上是一种"跨层专家混合"，但论文未与 MoE 架构进行对比或讨论组合可能性。
4. **注意力可视化仅定性**：关于 attention sink 行为的改变，缺乏定量分析（如 KV cache 压缩实验）。

## 六、适用场景

MoDA 最适合以下场景：

- **深度优先的模型架构**：当你希望用增加层数而非增加宽度来提升模型能力时，MoDA 是一个低开销的深度信息保护机制。
- **资源受限的中小模型**：在 1B-3B 参数量的模型上，MoDA 用仅 3.7% 的 FLOPs 开销换取 2%+ 的下游增益，性价比极高。
- **需要强层次特征的任务**：如数学推理、代码生成等需要多步逻辑链的任务，深度信息通道有助于保持早期推理步骤的关键信息。
- **Post-norm 架构探索**：如果你在尝试 post-norm 的深层网络，MoDA 可能是解决优化不稳定性的有效补充。

**暂不建议**的场景：超长上下文推理服务（缓存开销问题）、已高度优化的万亿参数工业级模型（需等 kernel 工程进一步成熟）。

---

**总结**：MoDA 是一个思路清晰、工程扎实的工作，将深度信息检索统一进标准注意力框架，并在硬件效率上做到了接近 FlashAttention-2 的水平。虽然在超大模型验证和推理效率上仍有待完善，但它为"如何让深层 Transformer 更好地利用深度信息"提供了一个非常有力的基线方案。对于关注 LLM 架构创新的读者，这篇论文值得深入阅读。
