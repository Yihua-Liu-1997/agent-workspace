## 🔍 背景与动机

构建统一的多模态架构是当前 AI 研究的核心目标之一。语言模型天然拥有语义丰富的离散 token，既支持理解也支持生成；但视觉模型却长期处于"分裂"状态——理解任务使用高维语义特征（768-1024 维），生成任务则依赖低维压缩 token（8-32 维）。这种割裂严重阻碍了真正统一的视觉-语言架构的实现。

离散视觉生成近年来备受关注，因为它使得视觉生成能够采用与语言模型相同的 token 预测范式。然而，现有的离散生成方法（如 MaskGIT、VAR 等）全部局限于低维潜在空间。如果直接使用高维表示 token 进行离散生成，会面临两个根本性挑战：

**第一，离散化难题**。传统的向量量化（VQ）方法在低维空间表现良好，但在 768 维以上的高维空间中，由于"维度灾难"，数据点极度稀疏，聚类效果急剧下降，固定大小的 codebook 根本无法充分覆盖高维特征空间。

**第二，建模难题**。即使成功离散化，一个 16×16×768 的表示张量包含 196,608 个离散 token。自回归生成需要 O(hwd) 步顺序生成，计算上完全不可行；而标准的离散扩散方法（如 MaskGIT）以空间位置为单位进行 mask，无法捕获同一空间位置内各维度之间的复杂依赖关系。

本文正是为了同时解决这两个挑战而生。

## 🧠 核心方法：Cubic Discrete Diffusion (CubiD)

CubiD 的核心思想可以用一句话概括：**将 h×w×d 的三维张量视为统一的"立方空间"，在其中进行细粒度的逐元素 mask 与预测。**

### 第一步：维度级量化（Dimension-wise Quantization）

论文采用维度级量化将冻结的预训练视觉编码器（DINOv2-B 或 SigLIP2-B）输出的高维连续特征离散化。与传统 VQ 对整个向量进行联合量化不同，维度级量化独立处理每个维度的连续值，将其映射为 L 个离散等级。这是一种无需训练的方法，可以直接应用于冻结的预训练特征。实验表明，DINOv2-B 在 L=8、SigLIP2-B 在 L=16 时即可达到与连续特征几乎一致的重建质量。

### 第二步：三维立方 mask 扩散

这是本文最核心的创新。CubiD 将离散化的 h×w×d 张量视为一个统一的建模空间，在训练时随机 mask 该张量中的任意元素（而非整个空间位置或整个维度），然后用 Transformer 通过双向注意力从未 mask 的部分预测被 mask 的 token。

具体而言：
- **Mask 比例采样**：从截断高斯分布中采样 mask 比例 r，均值 μ=1.0 偏向激进 mask，标准差 σ=0.10 为最优值
- **训练目标**：标准的交叉熵损失，预测被 mask 位置上的离散 token 类别
- **Mask token**：使用可学习的 mask token（优于固定值和随机采样）

### 推理过程

推理时从完全 mask 的张量出发，通过迭代式的"预测-去 mask"逐步生成完整图像。每次迭代中，模型并行预测所有被 mask 的 token，然后按余弦调度随机去 mask 一部分。关键优势在于：**无论特征维度 d 多大，生成步数 T 始终固定（通常 256-512 步），T ≪ hwd**。这使得高维离散生成在计算上变得可行。

### 架构设计

模型采用标准 Transformer 架构。每个空间位置的 d 个离散 token 被反量化为连续标量并拼接成 d 维特征向量，作为 Transformer 的一个输入 token。因此 Transformer 的序列长度仅为 h×w=256，与特征维度 d 无关。每个输出 token 通过 MLP 预测头产生 d×L 个 logits，同时预测该位置的所有维度。

## 🧪 实验设计

### 离散化验证

- **重建实验**：在 DINOv2-B（L=8, rFID=0.57）和 SigLIP2-B（L=16, rFID=0.69）上验证了维度级量化几乎无损地保持了连续特征的重建能力
- **理解实验**：在 LLaVA 框架下，用 GQA、TextVQA、POPE、MME 四个标准基准评测。维度级量化的 SigLIP2-DQ 与连续特征性能几乎一致（GQA: 63.1 vs 63.2），而 VQ 量化则出现严重退化（GQA: 54.9）

### 消融实验

论文进行了全面的消融研究：
- **Mask 策略**：Per-element（逐元素）显著优于 Per-dim（gFID=120.03）和 Per-spatial（gFID=22.22），验证了细粒度 mask 的必要性
- **推理步数**：64→256→512 步性能持续提升，512 步后饱和（gFID=5.25）
- **模型缩放**：946M→1.4B→3.7B 参数量持续改善（gFID: 5.25→4.91→4.68）
- **编码器泛化**：DINOv2 和 SigLIP2 均表现良好

### 主实验

在 ImageNet 256×256 类条件生成上与三类方法对比：低维离散扩散、低维自回归、高维离散模型。CubiD 是目前**唯一直接在原生高维表示空间（768 维）进行离散生成的方法**。

## 📊 结果与分析

### 核心结果

| 模型 | 潜在维度 | 参数量 | gFID↓ (无引导) | gFID↓ (有引导) |
|------|---------|--------|---------------|---------------|
| VAR | 32 | 2.0B | 2.16 | 1.97 |
| VFMTok-XXL | 12 | 1.4B | 1.95 | 2.19 |
| VFMTok-3B | 12 | 3.1B | 2.04 | 2.07 |
| **CubiD-L** | **768** | 946M | 2.38 | 2.37 |
| **CubiD-XL** | **768** | 1.4B | 2.06 | 2.04 |
| **CubiD-XXL** | **768** | **3.7B** | **2.02** | **1.88** |

CubiD-XXL 以 1.88 的 gFID（带 classifier-free guidance）取得了**离散生成方法的 SOTA**。值得注意的是，即使无引导条件下，CubiD-XXL 的 2.02 gFID 也超越了绝大多数方法。

### 关键发现

1. **高维 token 对引导的依赖更低**：表示 token 本身语义丰富，不需要强引导就能生成高质量图像，这与低维 VQ token 形成鲜明对比
2. **缩放行为优秀**：从 L 到 XL 到 XXL 持续改善，而 VFMTok 从 XXL 到 3B 反而略有退化，说明 CubiD 的方法更好地利用了增加的模型容量
3. **统一表示的有效性**：离散化后的高维 token 保持了原有的理解能力，同一套 token 可以同时服务理解和生成任务

## 💭 评价与思考

### 亮点

这篇论文解决了一个真正重要的问题：**视觉理解与视觉生成的 token 空间统一**。这不是一个小修小补的工作——它挑战了"生成必须用低维压缩空间"这一长期假设，开辟了通向真正统一多模态架构的新路径。

**方法设计优雅且直觉上合理**。将 h×w×d 张量视为三维立方空间进行逐元素 mask，这个想法简单但深刻。消融实验清楚地表明，per-dim 和 per-spatial 策略都严重失败，只有 per-element 策略才能有效捕获维度间和空间间的复杂依赖。这揭示了一个有趣的洞察：高维表示中，同一空间位置的不同维度之间也存在强耦合，不能简单解耦。

**实验验证全面**。从离散化质量（重建+理解）到消融实验再到主实验，逻辑链条完整。特别是"同一 token 既能理解又能生成"的验证，为统一架构提供了坚实的实证基础。

### 局限与思考

1. **分辨率受限**：目前仅在 ImageNet 256×256 上验证。16×16 的空间分辨率（由 DINOv2 的 patch size 决定）意味着 256×256 可能是当前方法的分辨率上限。向更高分辨率扩展需要应对更大的空间 token 数量或更复杂的编码器架构。

2. **推理效率**：虽然 T=256-512 步远优于自回归的 196,608 步，但与连续扩散模型（通常 20-50 步）或 VAR 等高效自回归方法相比仍有差距。每一步需要对所有被 mask 的 token 进行预测，虽然序列长度仅为 256，但每个位置的预测头需要输出 d×L=768×8=6144 个 logits，计算开销不小。

3. **token 数量的权衡**：196,608 个 token 意味着巨大的信息容量，但也带来了更高的生成难度。论文证明这是可行的，但这是否是最优的表示效率仍有待讨论。是否可以在保持语义丰富性的同时，通过更智能的 token 选择或压缩来减少冗余？

4. **统一架构的实际落地**：论文展示了理解和生成可以共享同一 token 空间，但距离真正的统一模型（如 Chameleon、Emu3 等端到端训练的模型）还有距离。如何在一个 Transformer 内同时处理理解和生成任务，需要更多探索。

### 更深层的影响

从更宏观的视角来看，这篇论文的真正价值在于它提出了一个"反压缩"的范式。过去几年，视觉生成的趋势是不断压缩：从像素到 VAE 潜在空间，再到更低维的 VQ token。而 CubiD 说的是：**也许我们不应该压缩，而应该直面高维语义空间的复杂性，用更聪明的算法来驾驭它。** 这种思路与 LLM 的发展形成了有趣的对照——语言模型一直使用高维语义 token，从未试图"压缩"语言。

## 🎯 适用场景

- **统一多模态架构研究**：为构建同时支持理解和生成的视觉模型提供新的技术路线
- **高保真图像生成**：768 维表示 token 携带丰富语义，适合对语义一致性要求高的生成任务
- **视觉 token 化**：为需要将视觉信息编码为离散 token 的应用（如视觉语言模型）提供新的 token 化方案
- **研究离散扩散的新范式**：CubiD 的三维 mask 策略为离散扩散在高维空间的应用提供了新的设计范式
