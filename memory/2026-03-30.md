## Copilot 接入 OpenClaw 的探索 🔌

### 需求
- 宝宝想让他的 Copilot 调用我的 session 当 sub agent，对工作区文件做研究等
- 探索了通过 CLI 命令实现的可能性

### 发现的三条路
1. **最优解：`openclaw mcp serve`**（MCP 服务器模式）
   - OpenClaw 作为 MCP Server，Copilot 作为 MCP Client 连接
   - 暴露工具：conversations_list, messages_read/send, events_poll/wait, attachments_fetch 等
   - **需要先升级**：当前版本 2026.3.13，需升级到 2026.3.28
   - 命令：`openclaw update` → `openclaw mcp serve`

2. **临时方案：`openclaw agent` CLI**（现在就能用）
   - `openclaw agent -m "问题" --json` 获取 JSON 回复
   - 缺点是无状态，但可通过 `--session-id` 保持连续会话

3. **不相关：`copilot-proxy` 插件** — 反方向，让 OpenClaw 用 Copilot 当模型 provider

### 关键信息
- Gateway 运行在 `ws://127.0.0.1:18789`，auth mode = token
- `openclaw update` 有可用更新：2026.3.28
