# JiT：无需训练的 DiT 空间加速框架

> arXiv: 2603.10744 | 2026年3月

---

## 一、背景与动机

扩散模型（Diffusion Models）近年来在图像生成领域取得了革命性突破，从 DALL-E 到 Stable Diffusion，再到最近的 FLUX 系列，生成质量不断提升。然而，高质量的代价是高昂的计算成本——一张高分辨率图像可能需要数十次神经网络评估（NFE），每次都要处理完整的空间 token 序列。

**核心问题**：Diffusion Transformers（DiT）将图像表示为离散 token 序列，通过 Transformer 架构逐步去噪。但这里存在一个被忽视的冗余——**空间冗余**：并非每个 token 在每个时间步都需要同等精度的计算。图像的平滑区域、背景区域，真的需要和细节区域一样的计算量吗？

现有加速方案各有局限：
- **时间域加速**（如减少采样步数）会损害质量
- **缓存方法**（如 TeaCache、TaylorSeer）受限于低 NFE baseline 的质量天花板
- **空间方法**（如 Bottleneck Sampling、RALU）依赖上采样操作，会引入伪影

作者提出的核心洞察是：**扩散生成本质上是"从粗到精"的过程**。早期阶段建立全局结构，后期阶段细化细节。如果能在早期只计算关键的"锚点 token"，通过智能插值推断其余部分，就能大幅节省计算量——这正是 JiT（Just-in-Time）框架的核心思想。

## 二、核心方法

JiT 框架包含三个紧密配合的技术组件，形成一个完整的加速体系：

### 2.1 空间近似生成 ODE（SAG-ODE）

传统扩散模型通过求解 ODE 来生成图像：
```
dy/dt = u_θ(y(t), t)
```
其中 `u_θ` 是神经网络预测的速度场，`y(t)` 是当前状态（所有 N 个 token）。

**JiT 的创新**：将全空间速度场分解为两个正交分量——活跃 token 的精确计算 + 非活跃 token 的智能推断。具体地，引入"增强提升算子" `Π_k`：

```
dy/dt = Π_k · u_θ(S_k^T · y(t), t)
```

其中 `S_k` 是选择矩阵，只选取 `m_k` 个活跃 token 进行计算。算子 `Π_k` 包含两部分：
1. **S_k · ũ**：将稀疏计算结果放回原位置
2. **I_k(ũ)**：通过插值推断非活跃 token 的速度

关键洞察：插值算子 `I_k` 使用**最近邻 + 掩码高斯模糊**的组合，而非传统的双线性插值。这是因为 DiT 的高维特征空间中，线性插值会导致特征漂移出有效流形，而最近邻能严格保持特征的统计特性，高斯模糊则平滑边界而不破坏锚点值。

### 2.2 重要性引导的 token 激活（ITA）

**问题**：应该选择哪些 token 作为锚点？

初始阶段使用确定性策略：从 2×2 网格中选取左上角 token，并强制包含所有边界 token（防止边缘伪影）。这确保了全局语义的稳定建立。

后续阶段则**动态选择**：基于速度场的方差计算"重要性图"。核心思想是——速度方差大的区域，意味着模型正在该区域进行重要的结构或细节生成，应该优先激活。这是一种"按需计算"的智能策略。

具体实现中，每个 token 的重要性分数是其速度向量在各维度上的方差，通过阈值筛选出 top-k% 作为下一阶段的锚点。

### 2.3 确定性微流（DMF）

当从一个阶段（稀疏 token）过渡到下一阶段（更密集 token）时，需要将现有状态扩展到新 token 上。简单插值会产生**噪声不匹配**问题——新 token 的噪声水平与流轨迹要求不符。

DMF 的解决方案：构造一个"目标状态"，该状态同时满足：
1. 与现有锚点 token 的一致性
2. 正确的噪声幅度（通过时间步 `t` 计算）

目标构造公式：
```
y* = Q_k(T_k · Φ_k(S_k^T · ŷ) + (1-T_k) · ε)
```

其中 `Φ_k` 是结构先验插值，`T_k` 是噪声调度，`ε` 是新采样的噪声。然后通过一个小型的确定性 ODE 求解步骤（微流），平滑地将状态推向目标。

这个设计确保了阶段转换的无缝性，避免了伪影的累积。

## 三、实验设计

### 3.1 实验设置

**基准模型**：FLUX.1-dev（当前 SOTA 的开源 DiT 文生图模型）

**对比方法**：
- 空间方法：RALU、Bottleneck Sampling
- 缓存方法：TeaCache、TaylorSeer
- 直接减少 NFE 的 vanilla baseline

**评估指标**：
- 生成质量：CLIP-IQA、ImageReward、HPSv2.1（人类偏好模型）
- 文本对齐：GenEval、T2I-CompBench

### 3.2 加速配置

JiT 采用 3 阶段调度：
- **4× 加速**（18 NFE）：35% → 62% → 100% token 激活
- **7× 加速**（11 NFE）：32% → 60% → 100% token 激活

时间步调度使用 Beta 分布（α=1.4, β=0.42）偏置早期阶段，确保全局结构的稳定建立。

## 四、结果与分析

### 4.1 定量结果

| 方法 | NFE | 加速 | CLIP-IQA | ImageReward | HPSv2.1 | GenEval |
|------|-----|------|----------|-------------|---------|---------|
| FLUX（50 NFE）| 50 | 1× | 0.614 | 1.004 | 30.39 | 0.657 |
| **JiT（4×）** | 18 | **4.24×** | **0.617** | **1.017** | 29.77 | **0.654** |
| **JiT（7×）** | 11 | **7.07×** | 0.540 | 0.975 | 29.02 | 0.646 |
| TeaCache（4×）| 28 | 4.10× | 0.600 | 0.964 | 29.68 | 0.649 |
| RALU（4×）| 18 | 4.13× | 0.549 | 0.856 | 29.83 | 0.633 |

**关键发现**：
1. JiT 在 4× 加速下，**几乎所有指标都超越或持平 50 NFE baseline**，这是其他方法无法企及的
2. 即使在激进的 7× 加速下，JiT 仍保持高质量，远超同等加速的其他方法
3. ImageReward 和 HPSv2.1 的高分表明 JiT 生成的图像更符合人类审美

### 4.2 定性结果

视觉对比揭示了 JiT 的独特优势：

**文字渲染**：在生成包含文字的图像时，JiT 能清晰渲染完整单词，而其他方法产生扭曲或残缺的文字。这是因为 JiT 的动态 token 选择能自动聚焦于高频细节区域。

**语义一致性**：其他方法在 7× 加速下常出现概念混淆（如将"猫"和"狗"的属性混合），而 JiT 保持清晰的语义边界。

**纹理细节**：JiT 避免了 RALU 等方法的上采样伪影（模糊、块状图案），图像保持自然锐利。

### 4.3 用户研究

20 名参与者对 50 个 prompt 生成的图像进行盲测偏好评估，总计 1000 票。结果显示：
- JiT vs FLUX（12 NFE）：**85.6%** 偏好 JiT
- JiT vs Bottleneck：**90.3%** 偏好 JiT
- JiT vs RALU（18 NFE）：**73.7%** 偏好 JiT

这证明了 JiT 在人类主观感知上的显著优势。

### 4.4 消融实验

作者系统验证了各组件的必要性：

| 变体 | HPSv2.1 | T2I-Comp |
|------|---------|----------|
| 完整 JiT | **0.269** | **0.373** |
| w/o SAG-ODE | 0.242 | 0.341 |
| w/o ITA | 0.265 | 0.367 |
| w/o DMF 目标 | 0.260 | 0.360 |

**分析**：
- 移除空间近似（SAG-ODE）导致灾难性下降——非活跃区域退化为无意义噪声
- 移除动态激活（ITA）引入固定网格无法覆盖的高频区域伪影
- 移除正确的 DMF 目标构造导致噪声不匹配，降低感知质量

## 五、评价与思考

### 5.1 方法优势

1. **无需训练**：直接应用于任何预训练 DiT，零额外成本
2. **即插即用**：不修改模型架构，只需调整推理流程
3. **理论扎实**：基于 ODE 理论的正交分解，有清晰的数学支撑
4. **效果显著**：在保持质量的前提下实现 4-7× 加速，实际可用

### 5.2 潜在局限

1. **超参敏感性**：token 稀疏度、阶段划分、时间步调度等需要针对不同模型调整
2. **极端加速的边界**：7× 加速下仍有质量损失，更高加速比可能不适用
3. **计算开销**：插值操作（特别是高斯模糊）引入额外开销，实际加速比可能低于理论值
4. **特定场景**：对极度依赖全局一致性的复杂构图，稀疏计算可能引入微妙的不协调

### 5.3 与相关工作对比

| 维度 | JiT | RALU | TeaCache |
|------|-----|------|----------|
| 加速来源 | 空间稀疏 | 空间稀疏 | 时间缓存 |
| 是否需要上采样 | 否 | 是 | N/A |
| 是否需要训练 | 否 | 否 | 否 |
| 质量保持 | 优秀 | 中等 | 受限于低 NFE |
| 通用性 | 高（图像+视频）| 中（图像）| 高 |

JiT 的核心优势在于**无需上采样**——通过 DMF 的确定性过渡，避免了传统空间方法引入伪影的根本原因。

## 六、适用场景

### 6.1 推荐应用

1. **实时交互式生成**：4× 加速下延迟从 25 秒降至 6 秒，接近交互体验
2. **批量图像生产**：内容创作、广告生成等需要大量图像的场景
3. **边缘设备部署**：降低计算需求，使高质量生成在消费级硬件上可行
4. **视频生成**：JiT 已在 HunyuanVideo-1.5 上验证，时空域加速可达 4-7×

### 6.2 不推荐场景

1. **医学/科学成像**：任何质量损失都不可接受的领域
2. **极端高分辨率**（>4K）：token 数量巨大，稀疏策略的收益递减
3. **需要确定性输出**：JiT 的随机 token 选择导致结果不可复现（可通过固定种子缓解）

### 6.3 未来方向

1. **自适应调度**：根据 prompt 复杂度自动调整加速比
2. **与量化结合**：4-bit 量化 + JiT 可实现 10×+ 加速
3. **多模态扩展**：音频、3D 生成等序列建模任务
4. **学习型调度**：用小型网络预测最优 token 选择策略

---

## 七、总结

JiT 代表了扩散模型加速的一个重要范式转变：从"均匀计算所有 token"到"按需计算关键 token"。通过 SAG-ODE 的数学框架、ITA 的智能选择、DMF 的平滑过渡，JiT 实现了 4-7× 的无损加速，为高质量图像生成的实际应用扫清了计算障碍。

其核心贡献不仅是技术方案本身，更在于揭示了一个深刻洞察：**扩散生成的本质是粗到精的过程，计算资源也应随之动态分配**。这种"及时计算"（Just-in-Time）的理念，很可能成为未来生成模型优化的通用范式。

---

**一句话总结**：JiT 通过空间稀疏化 + 智能插值 + 确定性过渡，在 FLUX.1-dev 上实现了 4-7× 无损加速，为 DiT 推理加速提供了训练无关、即插即用的优雅解决方案。
